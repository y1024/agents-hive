## D1：可嵌入 Agent 发布物

**Decision**

新增 `hive_embed_agent_configs`，不复用 `/api/v1/agents` 的 subagent 列表作为发布物。

字段基线：

- `id`
- `tenant_id`
- `domain_id`
- `name`
- `description`
- `base_prompt_key`
- `base_prompt`
- `default_tools`
- `allowed_origins`
- `context_schema`
- `ui_config`
- `enabled`
- `version`
- `published_at`
- `created_at`
- `updated_at`

单机或未启用多租户时，`tenant_id` 固定为 `default`。MVP 不做完整版本历史表，但 `version` 必须随发布配置递增，审计记录配置变更。

**Impact**

- `internal/store/*`
- `internal/api/embed_admin_handlers.go`
- `internal/api/embed_handlers.go`
- `frontend` 后续 Admin 配置入口
- SDK `getAgentConfig`

## D2：Token 签发模式

**Decision**

生产接入必须使用业务系统后端 token endpoint：

```text
Widget/SDK -> host backend token endpoint -> Hive Admin/服务端签发 API -> short-lived scoped token
```

Hive MVP 提供 Admin/服务端签发 API，不提供浏览器可直接调用的公开 token exchange。业务系统后端用服务端凭据调用 Hive 签发短期 token。

Token TTL：

- 浏览器 scoped token 默认 30 分钟。
- 最短 5 分钟，最长 60 分钟。
- server-side SDK 可使用最长 24 小时 token，但必须带 server scope，不能给浏览器。

PoC 可使用 `<script data-token="short_lived_scoped_token">`，但仅限短期 scoped token。

**Impact**

- `internal/auth/embed_token.go`
- `internal/api/embed_middleware.go`
- `internal/api/embed_admin_handlers.go`
- SDK `tokenProvider`
- Widget 初始化参数

## D3：Context 与 Env 边界

**Decision**

浏览器传入字段固定为：

- `business_context`: JSON 业务状态。
- `env`: string map，表达页面/流程环境。

二者都是不可信业务状态，只用于状态感知，不作为密钥、权限或后端环境变量。

校验规则：

- payload 最大 16KB。
- JSON 最大深度 4。
- key 最大 64 字符。
- 默认拒绝 key 名包含 `TOKEN`、`SECRET`、`PASSWORD`、`KEY`、`AUTHORIZATION`、`COOKIE`、`CREDENTIAL`。
- 如确需业务字段包含这些词，必须在 agent config 的 `context_schema` 中显式 allow。

**Impact**

- `internal/master/embed.go`
- `internal/master/prompt_builder.go`
- `internal/toolctx/context.go`
- SDK/Widget 类型定义
- 接入文档

## D4：Context 更新语义

**Decision**

- `hive_embed_sessions` 保存 session 最新 context/env。
- 每条消息处理前生成 context snapshot。
- snapshot 写入新增 `hive_embed_turns`，不塞进普通 `sessions.metadata`。
- `PATCH /embed/sessions/{id}/context` 使用 shallow merge。
- `sendMessage.context_update` 先 merge，再生成本轮 snapshot。
- `null` 删除 `business_context` 字段。
- `env` 只允许 string value；`null` 删除 env key。
- MVP 存完整 snapshot，不存 diff；审计事件记录 update 动作和 key 列表。

表结构基线：

```sql
CREATE TABLE IF NOT EXISTS hive_embed_turns (
    turn_id          TEXT PRIMARY KEY,
    session_id       TEXT NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
    tenant_id        TEXT NOT NULL,
    agent_id         TEXT NOT NULL,
    widget_id        TEXT NOT NULL DEFAULT '',
    token_id         TEXT NOT NULL DEFAULT '',
    context_snapshot JSONB NOT NULL DEFAULT '{}'::jsonb,
    env_snapshot     JSONB NOT NULL DEFAULT '{}'::jsonb,
    context_keys     JSONB NOT NULL DEFAULT '[]'::jsonb,
    created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

**Impact**

- `internal/store/*`
- `internal/api/embed_handlers.go`
- `internal/master/embed.go`
- SDK `updateContext`

## D5：工具凭据与业务工具

**Decision**

MVP 只支持：

- 平台 KB 三工具：`kb.doc.meta`、`kb.doc.structure`、`kb.section.text`。
- 明确注册且只读的业务工具，例如后续 `order_lookup`。

真实 ERP、工单、订单系统凭据必须在 Hive 后端或企业密钥系统配置，按 `tenant_id + agent_id + tool_name` 解析。浏览器不得传业务 API token。

每个业务工具必须声明 context binding，例如：

- `order_lookup` 只能访问当前 context snapshot 中的 `order_id`。
- `ticket_lookup` 只能访问当前 `ticket_id`。

MVP 不做通用“任意业务 API 调用工具”。

**Impact**

- `internal/toolctx/context.go`
- `internal/tools/*`
- `internal/router/tool_policy.go`
- `internal/master/react_processor.go`

## D6：工具权限解释

**Decision**

P2 提供最小只读 Admin 调试接口：

```
GET /api/v1/admin/embed/sessions/{id}/tool-decisions
```

返回：

- agent 默认工具。
- token 允许工具。
- session 收窄工具。
- 本轮模型可见工具。
- runtime 拒绝原因。

Widget 只展示用户友好错误，例如“当前助手未获授权执行该操作”，不展示内部 allowlist、policy 或工具 profile 细节。

**Impact**

- `internal/master/tool_visibility.go`
- `internal/master/react_processor.go`
- `internal/observability/*`
- `internal/api/embed_admin_handlers.go`

## D7：Stream 协议

**Decision**

Embed stream 对外使用统一 envelope：

```json
{
  "protocol_version": "embed.v1",
  "event_id": "evt_...",
  "type": "message",
  "session_id": "emb_...",
  "turn_id": "turn_...",
  "created_at": "2026-05-16T10:30:00Z",
  "payload": {}
}
```

首版事件类型固定为：

- `input_received`
- `message`
- `tool_call`
- `agent_status`
- `context_updated`
- `error`
- `session_closed`

MVP 不支持 `Last-Event-ID` 断线续传。重连后只接收新事件；历史消息通过 history API 获取。

**Impact**

- `internal/api/embed_stream.go`
- SDK `stream.ts`
- Widget renderer

## D9：租户隔离与身份映射

**Decision**

- Embed 请求映射为 synthetic user：`embed:<tenant_id>:<widget_id>`。
- 不在 `users` 表创建 external user。
- `tenant_id` 是强隔离字段，写入 session、turn、usage、audit、tool context。
- 如果现有 quota/accounting 要求 user 存在，扩展其 principal 模型支持 synthetic user，不反向污染后台用户体系。
- Admin UI 可显示 external user hash 或脱敏 ID，不显示明文 PII。

**Impact**

- `internal/auth/*`
- `internal/master/public_api.go`
- `internal/accounting/*`
- `internal/store/*`

## D10：Session 生命周期

**Decision**

- `embed_session.expires_at <= token.exp`。
- idle timeout 默认 60 分钟。
- 支持 `DELETE /api/v1/embed/sessions/{id}` 主动关闭。
- token revoke 后默认关闭该 token 创建的活跃 session，并发 `session_closed`。
- 未过期且同 token scope/origin/session 授权仍有效时，允许恢复历史 session。
- 过期 session 不能恢复，只能新建。

**Impact**

- `hive_embed_sessions`
- `internal/api/embed_handlers.go`
- `internal/api/embed_stream.go`
- SDK/Widget reconnect 逻辑

## D11：错误模型

**Decision**

Embed API 使用统一错误 envelope：

```json
{
  "error": {
    "code": "origin_denied",
    "message": "origin is not allowed",
    "request_id": "req_...",
    "retryable": false
  }
}
```

`code` 是稳定英文 snake_case，SDK/Widget 以 code 做逻辑判断。本地化由 SDK/Widget 根据 code 映射；服务端 message 可按 `Accept-Language` 后续增强，但 MVP 不依赖服务端双语。

首版必备 code：

- `missing_token`
- `invalid_token`
- `token_expired`
- `token_revoked`
- `origin_denied`
- `agent_denied`
- `scope_denied`
- `session_not_found`
- `session_expired`
- `rate_limited`
- `tool_denied`
- `tool_requires_approval`
- `context_invalid`

**Impact**

- `internal/api/*`
- `internal/errs/*`
- SDK `errors.ts`
- Widget error UI

## D13：MVP 范围

**Decision**

MVP 包含：

- Admin 创建 embed agent config。
- Admin/服务端签发短期 scoped token。
- `/api/v1/embed/sessions`
- `/api/v1/embed/sessions/{id}/messages`
- `/api/v1/embed/sessions/{id}/context`
- `/api/v1/embed/sessions/{id}/stream`
- TypeScript SDK。
- 最小 Widget。
- 工具 allowlist 和 runtime gate。
- 基础审计和 usage labels。
- KB 三工具的可见性、调用权限和 citation 事件透出。
- `embed.v1` 对 `asset_refs` 的协议兼容。

MVP 不包含：

- iframe。
- Go SDK。
- Webhook 回调。
- 复杂规则引擎。
- Agent 市场。
- 完整 Widget 管理后台。
- 长期浏览器 token。
- Widget 附件上传 UI。
- 原生移动 WebView 专项适配。

Widget MVP 做响应式 Web，兼容常见移动浏览器 viewport；不承诺原生 WebView 特性。UI 文案首版以 `zh-CN` 为主，保留 `en-US` 类型和后续扩展点。

注意：这里的“不包含 Widget 附件上传 UI”不等于可以不兼容资产。为了让业务侧最终使用 Hive 全部 AI 能力，SDK/Widget/stream v1 从一开始就必须能接收和展示 `citation`、`asset_refs`、`asset://` resolve 结果；只是用户主动上传附件的 Widget 控件可以后置。

**Impact**

- 所有 P0-P4 计划。

## D14：测试与验收标准

**Decision**

每个阶段必须保留安全回归：

- 未授权 origin 拒绝。
- 未授权 agent 拒绝。
- 未授权 tool 模型不可见且 runtime 不可执行。
- prompt injection 不能突破工具 gate。
- token revoke 立即生效。
- stream 不能订阅其他 session。
- context snapshot 可回放。

P4 Widget 必须使用浏览器自动化做桌面和移动端截图/QA，记录到 `.gstack/` 或等价 QA 报告。

**Impact**

- Go tests
- frontend tests
- SDK tests
- QA demo page
- `.gstack/` QA 记录

## D15：KB / Evidence 协同

**Decision**

- Hive Embed 的完整目标是把 Hive 的 KB/evidence 能力安全外露给业务系统；因此 KB P0/P1 不是可选优化，而是完整业务侧能力的硬依赖。
- Embed Agent 的 `default_tools` 若需要 KB，首版配置 `kb.doc.meta`、`kb.doc.structure`、`kb.section.text`。
- `kb_search` 只作为后续 vector/hybrid façade 或兼容入口，不作为 Embed MVP 的 KB 主路径。
- EmbedContext 不能携带授权性的 `kb_namespace`。
- KB namespace 只能由服务端 `KBBindingResolver` 根据 tenant、agent、domain、session template、session、user 解析。
- Embeddable agent config 不直接绑定 KB namespace；它引用 KB binding target，例如 `agent:<embed_agent_id>`。
- Stream/message metadata 必须保留 citations/evidence。
- Widget MVP 展示最小 citation 卡片；SDK 也把 citation 事件透给业务系统。
- 外部系统只传业务状态，不能传 `kb_namespace`、doc 权限或 evidence token 来扩大可见范围。

**Impact**

- `hive_embed_agent_configs.default_tools`
- `internal/kb/binding_resolver.go`
- `internal/api/embed_stream.go`
- SDK/Widget citation 渲染
- KB P0/P1 计划

## D16：对象存储协同

**Decision**

- 统一对象存储是“业务侧使用 Hive 全部 AI 能力”的硬依赖，不只是后续附件增强。
- Embed MVP 可以不做 Widget 附件上传 UI，但不能设计成不支持 asset。
- Widget 附件、Agent artifact、KB 图片、PDF/DOCX 转换资产必须复用 `internal/asset`。
- Stream 中出现 asset 引用时只暴露 `asset://` 或受控 asset ref。
- 前端展示前调用 asset resolve API。
- asset resolve 必须校验 tenant、session、agent、KB binding 或业务 ACL。
- Embed v1 至少保留并透传 `asset_refs`；是否展示 Agent artifact 下载链接可按 Widget UI 分阶段开启。

**Impact**

- `internal/asset/*`
- `internal/api/asset_handlers.go`
- `internal/api/embed_stream.go`
- Widget message renderer
- SDK event types
- `internal/asset` 统一对象存储层；原实施计划已归档到 `docs/计划与路线/归档/2026-05-16-统一对象存储层计划.md`
- KB P2 资产/PDF/DOCX 计划

## D17：Run 质量治理 / 观测事实源

**Decision**

- 不新增 `embed_runs` 作为主事实源。
- 使用 `session_id`、`turn_id`、`trace_id`、`span_id`、`tenant_id`、`agent_id`、`widget_id`、`token_id` 扩展现有 ExecutionRef。
- 对外 API 默认不暴露 `trace_id`；只返回 `request_id`、`session_id`、`turn_id`。
- Admin/debug API 可通过 `turn_id` 映射 trace。
- Widget 运行错误进入 audit/log；安全拒绝、tool denied、质量异常可进入 `agentquality.Event`。

**Impact**

- `internal/observability/*`
- `internal/agentquality/*`
- `internal/master/react_processor.go`
- `internal/api/embed_handlers.go`

## D18：业务域准入模型

**Decision**

- `agent_id` 只选择 embeddable agent config，不直接授予业务域工具。
- `hive_embed_agent_configs` 必须有 `domain_id`，默认 `generic`。
- 非 `generic` domain 必须经过 host-side `domainpolicy` 校验。
- EmbedContext 中的 `domain`、`workflow_stage` 只能作为状态提示，不作为 domain admission 输入。
- 若 Embed Agent 属于 `customer_service`，必须先通过客服业务域准入门禁。

**Impact**

- `internal/domainpolicy/*`
- `hive_embed_agent_configs`
- `internal/master/tool_visibility.go`
- `internal/router/capability_registry.go`

## D19：安全模型 / HITL

**Decision**

- Embed 模式下 `ToolPolicyDeny` 永远拒绝。
- Embed MVP 不执行 `ToolPolicyAsk` 工具。
- `ToolPolicyAsk`、high-risk 或需要后台审批的工具返回 `tool_requires_approval` 或 `tool_denied` stream 事件。
- 不把 approval request 透给业务系统 MVP。
- 后续如果要业务系统承接 HITL，必须定义独立 approval callback 协议、签名校验、超时和审计。

**Impact**

- `internal/master/react_processor.go`
- `internal/master/action_guard.go`
- `internal/api/embed_stream.go`
- SDK/Widget error handling

## D20：IM/EventRenderer/AG-UI 协同

**Decision**

- Embed stream handler 只做 EventBus 到 `embed.v1` envelope 的转译。
- 不改 EventBus 核心事件语义。
- Widget 只消费公开 envelope。
- 后台 WebSocket/IM renderer 继续消费内部事件。
- `embed.v1` 不强制采用 AG-UI 命名，但字段设计必须可映射到 AG-UI。

**Impact**

- `internal/api/embed_stream.go`
- `internal/streaming/websocket.go`
- `frontend/src/embed/sdk/stream.ts`
- Widget renderer

## D21：前端设计系统

**Decision**

- Widget 独立构建，不依赖后台 Chat store/router/auth。
- Shadow DOM 优先；若 Shadow DOM 不可用，使用严格命名空间 CSS。
- 可以复用 `ai-elements` 纯 leaf primitives 或其设计思想，但必须评估 bundle size 和 CSP。
- 禁止复用后台 Chat business shell。
- Widget 不写入 `internal/webui/dist`，输出到独立 embed build 目录。

**Impact**

- `frontend/src/embed/widget/*`
- `frontend/vite.config.embed-widget.ts`
