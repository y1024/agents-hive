# 补充计划：硬化、运营与验收（修正版）

> **执行优先级:** P5
> **目标:** 在 P0-P4 可用后，补齐成本、配额、生命周期、监控、文档、QA 和发布流程。

## 成本与配额

Embed session 的费用必须能归因到：

- tenant
- widget
- agent
- token
- session
- external user

需要扩展 usage/accounting 记录，至少增加：

```go
type EmbedUsageLabels struct {
    TenantID       string
    WidgetID       string
    AgentID        string
    TokenID        string
    SessionID      string
    ExternalUserID string
}
```

配额维度：

- 每 tenant 每日 token 数。
- 每 tenant 每日成本。
- 每 token 每分钟请求数。
- 每 token 并发 session 数。
- 每 session 最大消息数。
- 每 session 最大存活时间。

## Session 生命周期

策略：

- session `expires_at` 不得晚于 token `exp`。
- idle timeout 默认 30-60 分钟。
- 关闭 session 时广播 `session_closed` 事件。
- token revoke 后，应拒绝新请求，并可选择关闭该 token 创建的活跃 session。

需要测试：

- token 过期后不能继续发送消息。
- idle session 被清理。
- 关闭事件能被 SDK/Widget 收到。

## 监控指标

建议指标：

```text
embed_sessions_created_total{tenant_id,agent_id,widget_id}
embed_active_sessions{tenant_id,agent_id,widget_id}
embed_messages_total{tenant_id,agent_id,widget_id,status}
embed_stream_connections{tenant_id,agent_id,widget_id}
embed_token_denied_total{reason}
embed_origin_denied_total{origin}
embed_tool_denied_total{tool_name,reason}
embed_request_duration_seconds{route,status}
embed_llm_cost_usd_total{tenant_id,agent_id,widget_id}
```

告警：

- token denied 激增。
- origin denied 激增。
- stream 连接失败率升高。
- 单 tenant 成本超过阈值。
- tool denied 激增，可能是配置错误或攻击。

## 安全硬化

必须覆盖：

- Token 明文只返回一次。
- Token hash 存储。
- Origin 严格匹配。
- CORS preflight 不宽松放行。
- Browser 不使用长期 token。
- prompt 中外部 context 标记为不可信。
- context payload 有大小和深度限制。
- Widget 不使用未 sanitize 的 `dangerouslySetInnerHTML`。
- SDK 不把 token 放 URL query。

## QA 场景

端到端场景：

1. Admin 创建 embed agent config。
2. Admin 签发短期 scoped token。
3. Demo host 页面加载 Widget。
4. Widget 获取 token，创建 session。
5. Host 注入订单上下文。
6. 用户发送消息。
7. Agent 使用授权工具回答。
8. Agent 使用 KB 三工具返回带 citation 的回答，citation 通过 stream/SDK/Widget 可见。
9. 若回答涉及图片、PDF、附件或 artifact，stream 只透出 `asset://`/`asset_refs`，前端经 asset resolve API 展示。
10. 未授权工具被拒绝并审计。
11. 未授权 KB namespace 或 asset 被拒绝并审计。
12. token 过期后发送失败并提示刷新。

命令：

- `go test ./... -v`
- `cd frontend && npm run lint`
- `cd frontend && npm run build`
- `cd frontend && npm run build:embed-sdk`
- `cd frontend && npm run build:embed-widget`

如果引入浏览器 demo，需要补充截图或 QA 记录到 `.gstack/`。

## 文档交付

需要面向三类读者：

- 平台管理员：如何创建 embed agent、签发 token、配置 origin、查看审计。
- 业务系统后端：如何动态签发短期 token。
- 前端接入方：如何用 SDK/Widget 注入上下文、处理事件、处理错误。

文档必须明确：

- context/env 是不可信业务状态，不是 secret。
- Widget token 只允许短期 scoped token。
- 生产接入推荐 token endpoint。
- 工具权限由 Hive 后端配置，前端不能自行扩大。

## 最终验收

P5 完成时应满足：

- 一个真实业务页面可通过 Widget 嵌入 Agent。
- 一个 headless 脚本可通过 SDK 调用同一 Agent。
- 同一 Agent 在不同 context 下能感知业务对象变化。
- Embed Agent 能在业务侧使用 Hive KB 三工具，并展示服务端 evidence/citation。
- Embed stream/SDK/Widget 能兼容 `asset_refs`，通过 `internal/asset` resolve 展示授权资产。
- 未授权 agent、origin、tool、session 都被拒绝。
- 未授权 KB namespace、document、asset 都被拒绝。
- 管理员能看到 token/session/message/tool 的审计记录。
- 管理员能看到 KB evidence 和 asset resolve 的审计/质量事件。
- 成本和用量能按 tenant/widget/agent 汇总。
