# PPT / 脑图 / Mermaid / Artifact Studio 生成工具实施计划

> **For agentic workers:** 本文件是拆分后的索引入口。实现前必须先阅读 `00-交付契约与目标.md`，再按领域阅读对应子文档。禁止直接把任意 HTML 截图塞进 PPTX 作为“可下载 PPT”。核心原则仍是：每类 artifact 都必须走 `Spec -> Run -> Worker -> Asset -> Canvas Preview -> Download`。

**Goal:** 在 Chat 页面中生成可预览的 HTML slide deck、脑图、Mermaid、数据图表、表格、白板草图、公式和报告等可视化/办公 artifact，并提供可下载的 `.pptx`、`.svg`、`.png`、`.mmd`、`.mm.json`、`.xlsx`、`.csv`、`.tex`、`.pdf`、`.docx` 或 `.html` 产物。

**Architecture:** `DeckSpec` 是 presentation 唯一事实源；`DiagramSpec` 是 Mermaid/脑图/图解类产物的唯一事实源；后续扩展 `ChartSpec`、`TableSpec`、`WhiteboardSpec`、`EquationSpec`、`ReportSpec`。后端内置 `generate_ppt`、`generate_diagram`，并按路线增加 `generate_chart`、`generate_table`、`generate_whiteboard`、`generate_equation`、`generate_report`。所有能力复用统一 asset、权限、run store、Canvas preview/download 边界。

**Tech Stack:** Go 后端内置 MCP tool、React 19 + Zustand Canvas、统一 asset 服务、Node PPTX/diagram/chart/table exporter worker、PptxGenJS 4.0.1、Mermaid 11.15.0、@mermaid-js/mermaid-cli 11.15.0、markmap-lib/markmap-view 0.18.12、Vega-Lite 6.4.3、Vega-Embed 7.1.0、ECharts 6.0.0、ExcelJS 4.4.0、Excalidraw 0.18.1、tldraw 5.0.1、KaTeX 0.16.47、MathJax 4.1.2、服务端 Playwright 视觉兜底、Vitest/Go tests。

## 阅读顺序

1. 先读 `00-交付契约与目标.md`，确认不可妥协边界。
2. 实现 PPT 主链路时读 `02`、`04`、`05`、`06`、`07`。
3. 实现 Mermaid/Mindmap 时读 `03`、`04`、`05`、`06`、`07`。
4. 讨论 Chart/Table/Whiteboard/Equation/Report 时读 `03` 和 `09`，不要抢 M1/M2 主线资源。
5. 每次进入实现前，用 `07-分阶段实施计划.md` 的测试命令和验收用例做执行计划。

## 当前锁定结论

- M1/M2 主线：PPT 可编辑生成 + Mermaid Canvas/source/SVG 闭环。
- M3：Mindmap、ChartSpec、TableSpec、visual fallback、PNG/HTML 增强。
- M4：WhiteboardSpec、EquationSpec、ECharts 复杂业务图增强。
- M5：ReportSpec，输出 HTML/PDF/DOCX 并保留 provenance。
- PlantUML、Graphviz、draw.io、Figma、任意 React app artifact 默认不进入 M3/M4。
