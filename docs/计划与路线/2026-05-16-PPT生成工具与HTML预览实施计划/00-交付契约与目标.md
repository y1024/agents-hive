# 交付契约与目标

> 上级索引：[2026-05-16-PPT生成工具与HTML预览实施计划.md](../2026-05-16-PPT生成工具与HTML预览实施计划.md)

## 0. 12 分交付契约

这项能力的验收目标不是“能生成一个 PPT 文件”，而是把 PPT、脑图、Mermaid 等可视化产物生成变成 Hive 的一等内置工具能力。实现期间任何任务如果违反下面契约，应停止合并并回到计划修正。

### 0.1 不可妥协原则

- **同源事实源**：`DeckSpec` 是唯一事实源。HTML、PPTX、预览缩略图、notes 和后续编辑都必须从同一份 `DeckSpec` 派生。
- **可视化产物同样结构化**：脑图、Mermaid、流程图、时序图、图表、表格、白板、公式和报告不能只藏在 Markdown 代码块里。首发必须定义 `DiagramSpec`/`VisualizationArtifact` 元数据，扩展阶段必须定义各自 Spec/Run/Asset 合同，使预览、下载、历史恢复、权限和审计走同一套 artifact/asset 边界。
- **默认可编辑**：默认下载的 `.pptx` 必须由 PowerPoint 原生文本、形状、图片对象组成。不得把每页截图伪装成可编辑 PPTX。
- **首发只公开可编辑 PPTX**：图片版 PPTX 首发不对用户开放，只作为内部 QA/后续能力。后续如开放 `mode:"visual"`，文件名、UI、warnings 都必须标注“视觉版，文本不可编辑”。
- **权限事实服务端派生**：owner、user、domain、tenant、session、turn 等运行时事实只能从服务端 context 派生，不能来自模型输入。
- **HTML 不是信任边界**：模型不能传任意 HTML 给工具再转换 PPTX；工具生成的 HTML 也只用于预览，不作为 PPTX 输入源。
- **服务端稳定优先**：首发主链路必须能在服务端批量生成，不依赖用户浏览器状态、外部 CDN、Tailwind Play、Google Fonts 或 iframe 内脚本。
- **可观测可回滚**：每次生成都必须有 run id、validation report、worker report、asset tags、耗时、文件大小和失败分类。
- **首发非流式生成**：首发定义为“生成完成后立即预览和下载”，不承诺边生成边预览。流式/增量 DeckSpec 进入后续增强。

### 0.2 Definition of Done

首个对用户开放的版本必须同时满足：

1. Chat 中用户请求 PPT 时，模型调用 `generate_ppt`，工具返回 `kind:"presentation"` 结构化结果。
2. 右侧 Canvas 能打开 HTML deck，支持翻页、页码、notes、缩略图，且 UI 符合 Hive 控制台风格。
3. 下载按钮拿到 `.pptx`，Content-Type 正确，文件可由 PowerPoint/Keynote/WPS 打开。
4. 自动化测试 unzip PPTX，验证默认 editable 模式每页包含真实文本/形状对象，不是单张 full-slide PNG。
5. 错用户、错 session、错 purpose、错 asset format 的下载请求全部被拒绝。
6. Node worker 不可用、asset service 不可用、DeckSpec 非法、图片拉取失败时，工具返回结构化、可恢复或终止错误。
7. `DeckSpec` JSON、HTML、PPTX 三类产物都能追踪到同一个 `presentation_run_id`。
8. 所有借鉴/复制的开源代码、模板、token、布局契约都有 license 归属说明。
9. `presentation_runs` 数据表记录每次生成状态，不能只依赖 asset tags。
10. 首发不允许任意远程 HTTPS 图片；只允许已有 asset、受控内部图片和小体积 data URI。
11. 前端 presentation artifact 使用判别联合类型，不能把 metadata 留成裸 `Record<string, unknown>`。
12. 用户公开入口只显示 editable PPTX 下载；visual PPTX 不进入首发 UI。
13. Mermaid artifact 能在 Canvas 预览并下载 `.mmd` 和 `.svg`；渲染失败必须显示结构化错误，不允许空白。
14. Mindmap artifact 能在 Canvas 预览并下载源 JSON/Markdown 与 `.svg` 或 `.png`；节点内容和链接必须经过安全清洗。
15. 扩展 artifact 不允许绕开平台边界：`ChartSpec/TableSpec/WhiteboardSpec/EquationSpec/ReportSpec` 必须各自有 run id、asset tags、权限 purpose、Canvas renderer 和下载合同。
16. 数据图表必须能追踪数据来源；如果数据来自模型生成，工具结果必须标记 `data_provenance:"model_generated"`，不能伪装成真实业务数据。
17. 表格和图表必须可互相引用：`generate_chart` 能消费 `TableArtifact` asset，PPT/报告能嵌入 chart/table 的 SVG/PNG 或结构化 source。

### 0.2.1 可编辑 PPTX 质量标准

“可编辑”必须同时通过机器检查和人工检查。

机器检查：

- 每页至少有一个真实文本 shape，且标题文本能从 `ppt/slides/slideN.xml` 提取。
- 每页不能只有单张 full-slide 图片；允许背景图，但标题/正文仍必须是真实文本 shape。
- 图片元素必须是 PowerPoint 可选中图片对象，不是嵌在整页截图里。
- 色块、分割线、卡片背景使用 shape 对象。
- notes 若 DeckSpec 提供，必须进入 speaker notes；不支持时必须返回 warning，不能静默丢弃。

人工检查：

- 用 PowerPoint、Keynote 或 WPS 至少打开一份 Swiss fixture，确认标题/正文可直接编辑。
- 确认图片可选中和替换。
- 确认 16:9 画布不溢出、不重叠、中文换行可读。
- 确认 macOS 和 Windows 常见字体 fallback 下视觉不崩。

M2 不满足这些标准，不允许对普通用户开放。

### 0.3 阶段闸门

- **Gate 0：契约冻结**。冻结 `DeckSpec v1`、JSON Schema、layout registry、runtime context、`presentation_runs` store、asset policy、worker protocol、editable PPTX 验收规则。未完成不得写 UI。
- **Gate 1：端到端薄片**。用 3-4 个 Swiss layout 跑通 `DeckSpec -> HTML -> editable PPTX -> asset -> Canvas preview/download`，可 behind feature flag。
- **Gate 2：首发质量**。扩到 Swiss 8 个 layout，补全安全、权限、观测、移动端、i18n、部署自检。
- **Gate 3：增强能力**。再做 visual fallback 公开入口、PNG zip、更多模板、流式预览、自动排版和局部编辑。
- **Gate 4：可视化 Artifact 平台**。把 Mermaid、脑图、流程图、架构图统一到 `generate_diagram`、`DiagramSpec`、Canvas preview/download 和 asset 权限体系。
- **Gate 5：办公 Artifact Studio**。把 chart、table、whiteboard、equation、report 统一为 `generate_*` 工具族，复用 Spec/Run/Worker/Asset/Canvas/Download 平台边界。

## 1. 目标理解

用户期望的产品行为是：

1. 用户在 Chat 中说“帮我做一份 PPT”。
2. 系统生成一份 HTML slide deck，直接在右侧 Canvas 预览。
3. 用户点击下载时拿到 `.pptx` 文件，而不是 `.html` 或 `.md`。
4. PPTX 最好是可编辑对象：标题、正文、形状、图片应尽量是 PowerPoint 原生元素，而不是每页一张截图。
5. 用户说“帮我生成脑图 / Mermaid 流程图 / 时序图 / 架构图”时，也应在右侧 Canvas 预览，并能下载源文件与图片文件，而不是只能复制代码块。
6. 用户说“把这组数据做成趋势图 / 生成 Excel 表 / 画架构白板 / 输出公式 / 写成 PDF 报告”时，也应进入同一套 Canvas 预览和下载链路，而不是散落成普通 Markdown。

这不是寻找一个开源项目“一把梭”完成所有能力。正确方向是组合：`guizang-ppt-skill` 提供高质量视觉系统和版式纪律，`html-anything` 提供 Chat/HTML/deck 预览和视觉导出经验，PptxGenJS 提供可编辑 PPTX 对象生成，我们自己的 Hive 后端负责工具编排、资产、权限和服务端稳定性。
