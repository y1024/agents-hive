# 安全、权限与观测

> 上级索引：[2026-05-16-PPT生成工具与HTML预览实施计划.md](../2026-05-16-PPT生成工具与HTML预览实施计划.md)

## 5. 安全与权限

### 5.1 工具风险画像

在 `internal/router/capability_registry.go` 增加：

```go
"generate_ppt": {Domain: "media", Invocation: InvocationDirectTool, Risk: RiskLocalWrite, SideEffect: true},
"generate_diagram": {Domain: "media", Invocation: InvocationDirectTool, Risk: RiskLocalWrite, SideEffect: true},
```

理由：生成本地资产，有本地写入副作用，但不外发消息、不执行任意命令。

`generate_diagram` 同样只消费结构化 JSON 或受限 Mermaid 源码，不执行任意脚本，不读取本地文件，不访问外部网络。

### 5.2 HTML 安全

规则：

- 用户文本进入 HTML 前必须 escape。
- 不允许 DeckSpec 直接携带 raw `<script>`。
- 首发 Renderer 不生成翻页脚本；翻页、键盘、触摸、全屏由 React `DeckRenderer` 控制。
- 若后续需要页面内动画，Renderer 只能生成固定、内联、最小 JS，且必须走 `trustedGenerated` 路径。
- 首发不允许外部 `<script src>`。
- 图片只允许：
  - 已上传 asset URL
  - 后端受控内部图片 URL，并在生成时下载/转存为 asset
  - data URI 需限制大小和 mime

### 5.2.1 Mermaid / Mindmap 安全

规则：

- Mermaid 使用 `securityLevel:'strict'`，禁止 HTML label、外部脚本和不受控 click action。
- SVG 输出必须 sanitize：移除 `<script>`、`foreignObject>`、`on*` 属性、`javascript:` URL、外部 image href。
- Mermaid 源码最大 100KB；超过返回 `diagram_source_too_large`。
- MindMapSpec 节点数首发上限 300，深度上限 8，单节点文本 120 字符。
- MindMapSpec 不允许 raw HTML。Markdown outline 转换时只保留纯文本、层级和安全链接。
- 服务端 SVG/PNG 导出必须走受控 diagram worker，不允许模型传 Puppeteer/Playwright 参数。
- Diagram asset resolve 只允许 `source_kind=generated_diagram` 且 purpose 为 `diagram_preview/diagram_download/diagram_audit`。

### 5.3 PPTX 安全

规则：

- PPTX worker 只消费 JSON DeckSpec。
- 不读取任意本地路径。图片必须由 Go 侧 resolve 成受控临时文件或 base64。
- worker 超时默认 30s，可配置。
- 输出最大文件首发固定 50MB，超过则失败。

### 5.4 权限与多租户

资产 owner：

- `owner_scope=user`
- `owner_id=<auth user id>`

运行时事实：

- `generate_ppt` 不接受 `owner_id`、`owner_scope`、`session_id`、`domain_id` 作为模型输入。
- `RuntimeContext` 必须在 service 入口生成，并随同 `PresentationRun` 写入 logs/tags。
- asset upload 使用 runtime 中的 `OwnerScope/OwnerID`，不是请求体字段。
- asset resolve 使用 API auth user 派生 `ResolveContext`，前端只传 `uri/session_id/purpose`。

匿名/CLI 场景：

- 若无 auth user，使用 session/user fallback 需和现有 asset resolver 对齐；默认不允许生成需要浏览器下载的资产。
- 如果 assetService 不可用，工具应返回 recoverable error：“资产服务未初始化，无法生成可下载 PPTX”。

### 5.5 图片与外部资源安全

图片是 PPT 生成最容易出安全和稳定问题的输入。首发应把图片处理放在 Go 侧，不让 Node worker 自行访问网络。

允许来源：

- 已上传 `asset://...`。
- 当前 Hive 已允许的内部 `/api/...` 图片 URL，经服务端解析为 asset 或临时文件。
- 小体积 data URI：只允许 `image/png`、`image/jpeg`、`image/webp`，默认不超过 1MB。

首发禁止任意外部 HTTPS 图片 URL。M2/M3 若开启，必须通过统一 image resolver，并默认配置 `presentation.allow_remote_https_images=false`。

禁止来源：

- `file://`、本地绝对路径、相对路径。
- 任意外部 HTTPS URL，除非后续显式打开 `allow_remote_https_images` 并通过 allowlist。
- 内网 IP、localhost、link-local、metadata service。
- 任意 SVG 原文作为图片直接进入 PPTX。SVG 需要先 rasterize 或限制为后端生成的可信 SVG。

image resolver 要求：

- 使用允许域名策略，复用或扩展 `cfg.Tools.AllowedDomains`。
- DNS 解析后做私网 IP 拒绝，防 SSRF。
- 限制 Content-Length 和实际读取字节数。
- sniff MIME，不信任扩展名。
- 下载后转存为 presentation asset 或 run 临时文件，并记录 width/height/mime/bytes。
- 失败时返回 `image_resolution_failed`，由 schema/工具决定空图降级还是让模型修复。

### 5.6 PPTX 可编辑性验收

editable 模式输出后必须跑结构验收，不能只检查文件存在。

最低检查：

- zip 可打开。
- `[Content_Types].xml`、`ppt/presentation.xml` 存在。
- `ppt/slides/slideN.xml` 数量等于 `DeckSpec.slides.length`。
- 每页存在 `p:sp` 文本 shape 或 layout 定义允许的图文组合。
- 每页不允许只有一个覆盖全页的 `p:pic`，除非该页 layout 在 editable 模式明确允许背景图，且仍有真实文本 shape。
- 总文本提取结果包含每页标题。
- notes 如存在，应写入 PowerPoint speaker notes，或在首发明确“不导出 notes”并在 validation report 中 warning。

固定测试实现：

- 在 Go 测试中 unzip XML，统计 `p:sp`、`p:pic`、`a:t`。
- 对每个 fixture 断言标题文本在 `ppt/slides/slideN.xml` 中出现。
- M3 若开放 visual 模式，再增加反向断言：允许 full-slide PNG，但结果 `mode`、文件名、warnings 必须标明 visual。

### 5.7 观测与质量事件

每次生成必须记录：

- `presentation_run_id`
- `diagram_run_id`
- `session_id/turn_id/tool_call_id/user_id/owner_id/domain_id`
- `style/theme/mode/slide_count/layout_counts`
- `diagram_type/source_format/output_formats/node_count/edge_count/max_depth`
- `validation_ms/html_render_ms/pptx_export_ms/asset_upload_ms/total_ms`
- `diagram_validate_ms/diagram_render_ms/svg_sanitize_ms/diagram_asset_upload_ms`
- `html_bytes/pptx_bytes/image_count/image_bytes`
- `source_bytes/svg_bytes/png_bytes/html_bytes`
- `error_kind/recoverable/json_pointer`
- `worker_protocol_version/worker_version/pptxgenjs_version`
- `diagram_worker_protocol_version/diagram_worker_version/mermaid_version/markmap_version/playwright_version`

这些字段至少进入普通日志；若现有 agent quality/event 系统适合承载，也应同步一份失败质量事件，方便后续定位模型输出不稳定还是 exporter bug。

### 5.8 威胁模型与发布阻断项

按 STRIDE/OWASP 视角，首发必须把下列威胁作为发布阻断项处理：

| 威胁 | 入口 | 必须控制 |
|---|---|---|
| Spoofing | 模型伪造 `owner_id/session_id/domain_id` | tool schema 不接受这些字段；RuntimeContext 服务端派生 |
| Tampering | 篡改 asset proxy URL 为 attachment 或改 filename | `uri/expires/disposition/filename` 全部进 HMAC |
| Repudiation | 用户否认生成/下载 | `presentation_run_id`、download audit、run read audit 必须记录 |
| Information Disclosure | 错 owner/session 下载 PPTX | presentation asset resolver 按 owner/purpose/format/session 校验 |
| Denial of Service | 大 deck、大图、worker 卡死 | schema 限制、配额、worker timeout、output size、并发上限 |
| Elevation of Privilege | iframe 脚本读取同源数据 | 首发不允许 scripts，不加 `allow-same-origin` |
| SSRF | 图片 URL 访问内网 | 首发禁任意 HTTPS；后续 allowlist + DNS/IP 私网拦截 |
| Supply Chain | Node worker 依赖被污染 | package-lock、npm audit、只执行固定 worker path |
| Injection/XSS | 用户文本进入 HTML/PPTX | HTML escape，禁止 raw HTML/script，PPTX 只用结构化 text |
| Path Traversal | worker output/image path | Go 侧创建 temp dir，worker 只写指定 output path，禁止模型传路径 |
| SVG XSS | Mermaid/Mindmap SVG 预览或下载 | `securityLevel:'strict'`、SVG sanitize、禁止 `foreignObject/script/on*`、preview purpose 限制 |
| Link Injection | Mindmap 节点链接、Mermaid click | URL scheme allowlist、默认禁 Mermaid click action、`rel=noopener noreferrer` |
| Denial of Service | 巨大 mindmap tree、Mermaid 渲染爆炸 | source bytes、节点数、深度、SVG/PNG 输出大小、worker timeout |
| Supply Chain | Mermaid/Markmap/Playwright worker 依赖 | package-lock、npm audit、health check 记录版本、只执行固定 worker path |

发布阻断项：

- presentation asset resolver 没有负向测试，阻断发布。
- diagram asset resolver 没有负向测试，阻断发布。
- worker 仍能访问网络拉图，阻断发布。
- diagram worker 可访问外部网络或接受模型可控 CLI/Playwright 参数，阻断发布。
- iframe presentation 预览需要 `allow-same-origin`，阻断发布。
- 默认用户下载可能拿到 visual PPTX，阻断发布。
- Mermaid/Mindmap SVG sanitize 未覆盖 `<script>`、`foreignObject>`、`on*`、`javascript:`、外链 image，阻断发布。
- MindMapSpec 未限制节点数、深度和链接 scheme，阻断发布。
- 未提交 `package-lock.json` 或 `npm audit` high/critical 未处理，阻断发布。
- 未记录 license/NOTICE 来源，阻断发布。

### 5.9 License 与归属

借鉴两个本地项目时必须显式记录来源，避免后续复制模板或代码时丢归属。

- `guizang-ppt-skill`：MIT。可复用视觉 token、layout 规则和校验思路；若复制具体模板/CSS/脚本片段，必须在 `NOTICE` 或 `docs/架构设计/PPT生成工具架构.md` 记录来源路径和 MIT 归属。
- `html-anything`：Apache-2.0。可借鉴 deck viewer、per-slide HTML、export UX 和截图 readiness；若复制源码片段，必须保留 Apache-2.0 归属。
- PptxGenJS：MIT。记录版本 `4.0.1`、license 和升级策略。
- Mermaid / @mermaid-js/mermaid-cli：MIT。记录版本 `11.15.0`、license 和升级策略。
- markmap-lib / markmap-view / markmap-cli：MIT。记录版本 `0.18.12`、license 和升级策略。
- 首发新增 `internal/presentation/templates/*/template.json` 必须包含 `license`、`source_project`、`source_path`、`copied_code` 字段。
- 首发新增 `internal/diagram/templates/*/template.json` 如果存在，也必须包含 `license`、`source_project`、`source_path`、`copied_code` 字段。
- 如果只是“思路借鉴、重新实现”，`copied_code=false`；如果复制片段，`copied_code=true` 并记录文件和行段。
