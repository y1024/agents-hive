## 10. 锁定里程碑

### M0：契约与薄片准备

- DeckSpec v1 + JSON Schema。
- DiagramSpec v1 + JSON Schema。
- RuntimeContext + PresentationRun 合同。
- DiagramRun 合同。
- Worker protocol + asset policy。
- Diagram worker protocol + diagram asset policy。
- Swiss 3-4 个 layout fixture。
- Mermaid/Mindmap golden fixtures。
- `presentation_runs` store/migration 设计。
- `diagram_runs` store/migration 设计。
- 计划文档进入可提交路径或 `.gitignore` 例外。
- 冻结非流式首发、editable-only、远程 HTTPS 图片关闭、visual 不公开。
- 冻结 Mermaid M2、Mindmap M3 或明确前移的产品决策。

状态：内部开发闸门，不对普通用户开放。

### M1：端到端薄片

- DeckSpec v1。
- Swiss HTML renderer。
- Node worker 支持 3-4 个 Swiss layout 的 editable PPTX。
- `generate_ppt` 返回 DeckSpec/HTML/PPTX 三类 asset。
- 前端可从工具结果打开 Canvas 预览。
- 前端可下载 editable PPTX。
- XML 验收证明每页有真实文本对象。
- `presentation_runs` 记录 succeeded/failed/running 状态。
- 同步超时可返回 running，前端显示生成中。
- Mermaid `generate_diagram` 薄片可生成 source/SVG asset。
- 前端可从 diagram 工具结果打开 Canvas 预览 Mermaid，并下载 `.mmd/.svg`。
- `diagram_runs` 记录 succeeded/failed/running 状态。

状态：可 behind feature flag 内测。

### M2：首发可用

- 支持 Swiss 8 个 layout。
- 支持 Mermaid Canvas 预览和 `.mmd/.svg` 下载。
- presentation asset 权限策略和负向测试完整。
- diagram asset 权限策略和负向测试完整。
- 图片 asset 转存、小体积 data URI 和内部图片支持完整；任意外部 HTTPS 图片仍默认关闭。
- i18n、移动端、错误态、部署 health check 完整。
- Mermaid SVG sanitize、source 大小限制、错误态和历史恢复完整。
- 人工 PPTX 质量验收通过：PowerPoint/Keynote/WPS 至少一种可打开并编辑。
- `go test` / `npm test` / `npm run lint` / `npm run build` 通过。

### M3：质量和体验增强

- 自动 layout 推荐。
- HTML/PPTX screenshot diff。
- 更多 Swiss layout。
- Editorial Style A。
- visual PPTX 和 PNG zip。
- Mindmap Canvas 预览和 `.mm.json/.md/.svg` 下载。
- Mermaid/Mindmap PNG 导出。
- Diagram 自包含 HTML 下载。
- ChartSpec / `generate_chart`：Vega-Lite source、Canvas 预览、`.vl.json/.svg` 下载，PNG behind config。
- TableSpec / `generate_table`：Canvas 表格预览、`.xlsx/.csv/.json` 下载，Excel/CSV 注入防护。
- PPT 能引用 chart/table asset，并在 warnings/metadata 中保留 provenance。
- “修改第 N 页”局部编辑。
- 流式/增量预览。
- 任意外部 HTTPS 图片 allowlist 支持。

### M4：白板和公式

- WhiteboardSpec / `generate_whiteboard`：Excalidraw source、Canvas 白板预览、`.excalidraw/.svg/.png` 下载。
- EquationSpec / `generate_equation`：TeX source、Canvas 公式预览、`.tex/.svg/.png` 下载。
- ECharts 复杂业务图表增强，前提是 ChartSpec 白名单和安全策略完成。
- tldraw 只做 SDK/license/bundle size PoC，不默认替代 Excalidraw。

### M5：报告交付

- ReportSpec / `generate_report`：引用 PPT、chart、table、diagram、equation asset，输出 `.html/.pdf/.docx`。
- PDF/DOCX 引擎完成单独 PoC 后进入实现。
- 报告导出保留所有 source run ids 和 data provenance。

## 12. 推荐实施顺序

1. 先做 Phase 0 契约冻结，不碰 UI。
2. 先处理计划文档可追踪路径，确保后续评审能进版本库。
3. 做 `DeckSpec`、schema、validator、fixtures。
4. 做 `DiagramSpec`、schema、validator、fixtures，先锁 Mermaid/Mindmap 合同。
5. 做 RuntimeContext、PresentationRun/DiagramRun store、asset tags、presentation/diagram asset resolver。
6. 做 HTML renderer 和 PPTX worker 的 3-4 layout 薄片。
7. 做 diagram worker 的 Mermaid source/SVG 薄片。
8. 做 Go service 编排和 `generate_ppt` / `generate_diagram` 工具注册。
9. 做前端 presentation artifact、DeckRenderer、预览和 PPTX 下载。
10. 做前端 diagram/mindmap artifact、DiagramRenderer、MermaidRenderer、MindMapRenderer、源码/SVG 下载。
11. 扩 Swiss 8 layout，补图片 resolver、权限负测、部署 health check。
12. 做端到端 QA、移动端、i18n、可观测。
13. 做 ChartSpec/TableSpec 和 `generate_chart/generate_table`，建立数据可信底座。
14. 做 PPT 引用 chart/table asset，保留 data provenance。
15. 再做 visual fallback、Mindmap M3、PNG/HTML 导出、更多版式、流式预览和局部编辑。
16. 做 Whiteboard/Equation M4，最后做 Report M5。
