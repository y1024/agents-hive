# IM 统一抽象与平台路由整改 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans or an equivalent task-by-task execution workflow. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立统一、可插拔、平台感知的 IM Agent 能力层，根治“用户要求微信/企微/钉钉发送却被路由到飞书工具”的系统性问题。

**Architecture:** 模型只面向统一 IM Agent API；平台差异收敛到 adapter/capability registry；路由、契约、工具可见性、审计和测试都携带 `platform` 维度。`feishu_api` 保留为飞书文档/表格/审批等域工具，但 IM 外发逐步迁移到统一入口。

**Tech Stack:** Go, `internal/imctx`, `internal/channel`, `internal/tools`, `internal/master`, `internal/router`, PostgreSQL store, table-driven Go tests.

## 1. 事故证据

### 1.1 飞书第 6 轮才发出的真实链路

会话 `fe8d9552-70ab-4d42-8741-3b7ab0996c6a` 的关键顺序：

1. `search_contacts("郭松")` 只暴露 `user_id=afde2a69`，没有稳定引导模型使用 `open_id`。
2. 模型把 `user_id` 塞进 `chat_id` 调 `feishu_api.send_message`。
3. 飞书返回 `invalid receive_id`。
4. 模型误以为必须让用户提供 `chat_id`，触发不必要澄清。
5. 用户说不知道后，模型才 `get_user_info(user_id)` 拿到 `open_id=ou_e48...`。
6. 第 6 轮使用 `ou_e48...` 发送成功。
7. 成功后仍继续工具调用，原因是外部发送 `tool_choice=required` 没有基于“本轮已有成功发送证据”放松，同时契约证据没有用后续成功覆盖早先失败。

这部分已通过近期修复缓解，但它暴露出同一个根因：IM 发送没有统一 recipient 语义，契约也没有平台维度。

### 1.2 微信请求被错路由到飞书工具

最新会话 `6f12cc94-5e48-47f3-8951-d2c6b73b4c84`：

- 用户输入：`给微信用户也发一条，你自己选择跟谁发都行`
- 路由日志：
  - `intent_kind = external_write`
  - `allowed_tools = ["feishu_api"]`
  - `candidate_names = ["glob", "feishu_api", "grep", "question"]`
  - `final answer did not satisfy user intent`
- 结论：
  - 平台词“微信”没有进入结构化 intent。
  - `CapabilityExternalSend` 粗粒度授权让飞书工具满足了“任意外部发送”。
  - `send_im_message` 未被召回或未通过路由可调用。

这不是微信单点问题；企微、钉钉都会遇到同类问题。

### 2.2 结构性缺口

1. **工具层缺统一 IM Agent API**
   - `send_im_message` 只负责发送，不提供联系人/会话发现。
   - `feishu_api` 同时承担飞书专用业务 API 和 IM 发送能力。
   - 模型必须自己猜该调 `feishu_api` 还是 `send_im_message`。

2. **路由层缺平台维度**
   - `IntentFrame` 有 `AllowedDomainsHint`，但当前 external-send 恢复逻辑没有稳定填充平台。
   - `BuildRouteDecision` 只看 `CapabilityExternalSend`，导致 `feishu_api` 可以满足微信/企微/钉钉请求。

3. **召回层存在飞书偏置**
   - `pruneGenericIMWhenFeishuDomainEntryRecalled()` 只要召回到 `feishu_api` 就删除 `send_im_message`。
   - 这对“飞书用户”合理，但对“微信用户/企微用户/钉钉用户”是错误压制。

4. **契约层只判断“是否发送”，不判断“发到哪个平台”**
   - `intent_contract.go` 的 external send evidence 只看 send attempt/content/recipient。
   - 飞书发送成功不能满足微信发送意图。

5. **平台 capability 没有结构化表达**
   - 现在是 `external.send` 一个能力。
   - 缺 `external.send.feishu`、`external.send.wechatbot`、`external.send.wecom`、`external.send.dingtalk`。

6. **recipient discovery 形态不统一**
   - 飞书可 `search_contacts/get_user_info`。
   - 微信可列已有 `wechat_conversations`，但没有给 Agent 的统一只读查询工具。
   - 企微/钉钉当前没有按姓名查人的 Agent 工具，若没有 chat_id 应该要求澄清或说明能力边界。

## 3. 产品与技术原则

1. **模型只看统一能力，不看平台实现细节**
   - 模型应调用 `im_api` 或受限的统一入口，而不是在多个平台私有工具之间猜。

2. **平台 adapter 可插拔**
   - 新增 Slack/Telegram/短信/邮件时，只实现 adapter 和 capability metadata，不改主循环。

3. **平台语义进入 intent、route、contract、audit**
   - 用户说微信，路由只能放微信能力。
   - 用户说企微，飞书成功不算完成。

4. **统一 ID 语义**
   - 对模型暴露 `recipient_id` / `conversation_id` / `platform`。
   - 平台内部的 `open_id/user_id/chat_id/peer_wxid/context_token` 由 adapter 解析。

5. **owner 与租户边界由宿主派生**
   - `wechatbot.owner_user_id` 只能来自 `auth.UserFrom(ctx)`。
   - 模型输入里出现 `owner_user_id` 必须忽略或拒绝。

6. **非必要不审批**
   - 普通 IM 文本发送不走 HITL 审批。
   - 删除、写表、创建审批、危险管理操作才走审批。

7. **能力不足必须明确反馈**
   - 企微/钉钉若没有联系人搜索工具，不允许模型按姓名盲发。
   - 微信若没有可发送会话，不允许模型编造联系人；应该列出可发送会话或要求对方先发消息。

## 4. 目标架构

### 4.1 统一 Agent 工具：`im_api`

新增一个面向模型的统一 IM 工具，首批 action：

```json
{
  "action": "search_recipients | list_recent_conversations | resolve_recipient | send_message",
  "platform": "feishu | wechatbot | wecom | dingtalk",
  "query": "郭松",
  "recipient_id": "平台中立 recipient id",
  "conversation_id": "平台中立 conversation id",
  "content": "消息内容",
  "dry_run": false
}
```

模型期望流程：

- 用户说“给飞书用户郭松发消息”
  - `im_api.search_recipients(platform=feishu, query=郭松)`
  - `im_api.resolve_recipient(platform=feishu, recipient_id=...)`
  - `im_api.send_message(platform=feishu, recipient_id=..., content=...)`
- 用户说“给微信用户也发一条”
  - `im_api.list_recent_conversations(platform=wechatbot)`
  - 只要用户没有明确 recipient/conversation_id，就必须 `question` 让用户确认目标；即使只有一个 `can_send=true` 会话也不能自动发。
  - 没有可发送会话则说明阻塞：当前没有可发送微信会话，请让对方先发一条消息或在 Web 端提供已归属当前 owner 的 peer wxid。
- 用户说“给企微用户张三发”
  - 若 adapter 不支持 search，则 `im_api.search_recipients` 返回结构化 unsupported。
  - 模型必须请求 chat_id 或提示当前未接入企微联系人搜索。
- 用户直接给 `chat_id/conversation_id`
  - `im_api.send_message(platform=..., conversation_id=..., content=...)`
- 用户说“飞书和微信都发”
  - Phase 1 不做原子 fan-out。
  - 系统必须先 `question` 让用户确认发送顺序/目标，或将其拆成两个单平台子任务分别执行，每个子任务只有一个 `AllowedDomainsHint`。

### 4.2 Adapter 接口

新增包建议：`internal/imcore`

```go
package imcore

import "context"

type Platform string

type Capability string

const (
	CapabilitySearchRecipients              Capability = "search_recipients"
	CapabilityListRecentConversations       Capability = "list_recent_conversations"
	CapabilitySendText                      Capability = "send_text"
	CapabilitySendImage                     Capability = "send_image"
	CapabilityRequiresOwnerContext          Capability = "requires_owner_context"
	CapabilityRequiresConversationContext   Capability = "requires_conversation_context"
)

type CallerScope struct {
	OwnerUserID string
	TenantKey   string
	TraceID     string
}

type Recipient struct {
	Platform       Platform
	ID             string
	DisplayName    string
	Kind           string // user | group | conversation
	ExternalIDType string // open_id | user_id | chat_id | peer_wxid | conversation_id
	CanSend        bool
	SendState      string // ready | no_context | unsupported | ambiguous
	Metadata       map[string]string
}

type SendTarget struct {
	Platform       Platform
	RecipientID    string
	ConversationID string
	Content        string
	DryRun         bool
}

type SendResult struct {
	Platform       Platform
	TargetID       string
	TargetKind     string
	MessageID      string
	Delivered      bool
}

type Adapter interface {
	Platform() Platform
	Capabilities() []Capability
	SearchRecipients(ctx context.Context, scope CallerScope, query string, limit int) ([]Recipient, error)
	ListRecentConversations(ctx context.Context, scope CallerScope, limit int) ([]Recipient, error)
	ResolveRecipient(ctx context.Context, scope CallerScope, input RecipientLookup) (Recipient, error)
	SendMessage(ctx context.Context, scope CallerScope, target SendTarget) (SendResult, error)
}

type RecipientLookup struct {
	Query          string
	RecipientID    string
	ConversationID string
}
```

`CallerScope` 的唯一来源是 `imcore.Service` 从 `auth.UserFrom(ctx)` 和 `toolctx.ToolContext` 派生。模型输入里的 `owner_user_id` 一律忽略；`WechatBotAdapter` 收到 `scope.OwnerUserID == ""` 必须直接拒绝。adapter 内部再次用 `scope.OwnerUserID` 校验 `recipient_id/conversation_id/peer_wxid` 归属，不能信任模型传入的任何目标 ID。

### 4.3 平台 adapter 职责

| 平台 | Adapter | 支持能力 | 限制 |
|---|---|---|---|
| 飞书 | `FeishuAdapter` | 搜联系人、解析 user_id/open_id、单聊/群聊发送 | 复用现有 `FeishuToolProvider`；飞书文档/审批仍留在 `feishu_api` |
| 微信个人号 | `WechatBotAdapter` | 列已有会话、发送到可发送会话 | 必须 owner-scoped；需要 `wechat_conversations.can_send=true`；不能按姓名全局搜索 |
| 企业微信 | `WeComAdapter` | 发送到已知 chat_id | 当前未实现通讯录搜索；命名用户必须澄清 |
| 钉钉 | `DingTalkAdapter` | 发送到已知 chat_id | 当前未实现通讯录搜索；命名用户必须澄清 |

### 4.4 兼容策略

- `send_im_message` 暂时保留，内部改为调用 `imcore.Service.SendMessage`。
- `feishu_api.send_message` 短期保留，内部可复用 `FeishuAdapter` 的解析逻辑。
- 模型可见工具优先逐步切换：
  1. Phase 1：新增 `im_api`，仍保留旧工具。
  2. Phase 2：平台外发优先召回 `im_api`。
  3. Phase 3：`send_im_message` 从模型默认路径降级为兼容工具。
  4. Phase 4：`feishu_api` 的消息发送 action 标为 legacy；飞书业务能力不动。

```go
func externalSendPlatformsFromQuery(query string) []string {
	q := strings.ToLower(strings.TrimSpace(query))
	var out []string

			if existing == platform {
				return
			}

	}

		add("dingtalk")
	}

}
```

```go
func ensureExternalSendCandidates(recalls []tools.ToolRecallHit, catalog []mcphost.ToolDefinition, intent router.IntentFrame, score float64) []tools.ToolRecallHit {
	if !isExplicitExternalSendIntent(intent) {
		return recalls
	}
	platforms := stringSet(intent.AllowedDomainsHint)
	preferGenericIM := platforms["wechatbot"] || platforms["wecom"] || platforms["dingtalk"]
	preferFeishu := len(platforms) == 0 || platforms["feishu"]
	seen := make(map[string]bool, len(recalls))
	for _, recall := range recalls {
		if name := strings.TrimSpace(recall.Tool.Name); name != "" {
			seen[name] = true
		}
	}
	if score <= 0 {
		score = 1
	}
	for _, tool := range catalog {
		name := strings.TrimSpace(tool.Name)
		if name == "" || seen[name] {
			continue
		}
		if preferGenericIM && name != "send_im_message" {
			continue
		}
		if !preferGenericIM && !preferFeishu {
			continue
		}
		if !profileSupportsExternalSend(router.InferToolProfile(tool, router.ProfileHint{})) {
			continue
		}
		recalls = append(recalls, tools.ToolRecallHit{Tool: tool, Score: score})
		seen[name] = true
	}
	return recalls
}
```

```go
package imcore

import (
	"context"
	"fmt"
	"strings"

	"github.com/chef-guo/agents-hive/internal/auth"
	"github.com/chef-guo/agents-hive/internal/toolctx"

type Service struct {
	adapters map[Platform]Adapter
}

func NewService(adapters ...Adapter) *Service {
	s := &Service{adapters: map[Platform]Adapter{}}
	for _, adapter := range adapters {
		if adapter == nil || adapter.Platform() == "" {
			continue
		}
		s.adapters[adapter.Platform()] = adapter
	}
	return s
}

func (s *Service) Adapter(platform Platform) (Adapter, error) {
	if s == nil {
		return nil, fmt.Errorf("im service not configured")
	}
	adapter := s.adapters[platform]
	if adapter == nil {
		return nil, fmt.Errorf("im platform %s not configured", platform)
	}
	return adapter, nil
}

func (s *Service) callerScope(ctx context.Context, platform Platform) (CallerScope, error) {
	scope := CallerScope{}
	if user := auth.UserFrom(ctx); user != nil {
		scope.OwnerUserID = user.ID
		scope.TenantKey = user.ID
	}
	if tc := toolctx.GetToolContext(ctx); tc != nil {
		scope.TraceID = tc.TurnIDOrTraceID()
	}
	if platform == PlatformWeChatBot && strings.TrimSpace(scope.OwnerUserID) == "" {
		return CallerScope{}, fmt.Errorf("wechatbot requires authenticated owner")
	}
	return scope, nil
}

func (s *Service) SearchRecipients(ctx context.Context, platform Platform, query string, limit int) ([]Recipient, error) {
	adapter, err := s.Adapter(platform)
	if err != nil {
		return nil, err
	}
	if strings.TrimSpace(query) == "" {
		return nil, fmt.Errorf("query is required")
	}
	if limit <= 0 {
		limit = 10
	}
	scope, err := s.callerScope(ctx, platform)
	if err != nil {
		return nil, err
	}
	return adapter.SearchRecipients(ctx, scope, query, limit)
}

func (s *Service) ListRecentConversations(ctx context.Context, platform Platform, limit int) ([]Recipient, error) {
	adapter, err := s.Adapter(platform)
	if err != nil {
		return nil, err
	}
	if limit <= 0 {
		limit = 10
	}
	scope, err := s.callerScope(ctx, platform)
	if err != nil {
		return nil, err
	}
	return adapter.ListRecentConversations(ctx, scope, limit)
}

func (s *Service) ResolveRecipient(ctx context.Context, platform Platform, input RecipientLookup) (Recipient, error) {
	adapter, err := s.Adapter(platform)
	if err != nil {
		return Recipient{}, err
	}
	scope, err := s.callerScope(ctx, platform)
	if err != nil {
		return Recipient{}, err
	}
	return adapter.ResolveRecipient(ctx, scope, input)
}

	adapter, err := s.Adapter(target.Platform)
	if err != nil {
		return SendResult{}, err
	}
	if strings.TrimSpace(target.Content) == "" {
		return SendResult{}, fmt.Errorf("content is required")
	}
	scope, err := s.callerScope(ctx, target.Platform)
	if err != nil {
		return SendResult{}, err
	}
	return adapter.SendMessage(ctx, scope, target)
}
```

```go
package tools

import (

type imAPIInput struct {
	Action         string `json:"action"`
	Platform       string `json:"platform"`
	Query          string `json:"query,omitempty"`
	RecipientID    string `json:"recipient_id,omitempty"`
	ConversationID string `json:"conversation_id,omitempty"`

	DryRun         bool   `json:"dry_run,omitempty"`

func RegisterIMAPITool(host *mcphost.Host, logger *zap.Logger, service *imcore.Service) {
	schema, _ := json.Marshal(map[string]any{
		"type": "object",
		"properties": map[string]any{
			"action": map[string]any{
				"type": "string",
				"enum": []string{"search_recipients", "list_recent_conversations", "resolve_recipient", "send_message"},
				"description": "IM 操作",
			},
			"platform": map[string]any{
				"type": "string",
				"enum": []string{"feishu", "wechatbot", "wecom", "dingtalk"},
				"description": "目标 IM 平台",
			},
			"query": map[string]any{"type": "string", "description": "联系人或会话搜索关键词"},
			"recipient_id": map[string]any{"type": "string", "description": "im_api 返回的平台中立 recipient id"},
			"conversation_id": map[string]any{"type": "string", "description": "im_api 返回的平台中立 conversation id 或平台 chat id"},
			"content": map[string]any{"type": "string", "description": "要发送的纯文本内容"},
			"limit": map[string]any{"type": "integer", "description": "返回数量，默认 10"},
			"dry_run": map[string]any{"type": "boolean", "description": "为 true 时只校验目标和权限，不真实发送"},
		},
		"required": []string{"action", "platform"},
	})

	host.RegisterTool(mcphost.ToolDefinition{
		Name: "im_api",
		Description: "统一 IM 工具。用于飞书、个人微信、企业微信、钉钉的联系人/会话发现和消息发送。按平台能力执行；不支持的能力会返回明确错误。",
		InputSchema: schema,
	}, func(ctx context.Context, input json.RawMessage) (*mcphost.ToolResult, error) {
		var params imAPIInput
		if err := json.Unmarshal(input, &params); err != nil {
			return errorResult("解析参数失败: " + err.Error()), nil
		}
		if service == nil {
			return errorResult("im_api 未配置 IM service"), nil
		}
		platform := imcore.Platform(params.Platform)
		switch params.Action {
		case "search_recipients":
			items, err := service.SearchRecipients(ctx, platform, params.Query, params.Limit)
			if err != nil {
				return errorResult(err.Error()), nil
			}
			return jsonToolResult(items), nil
		case "list_recent_conversations":
			items, err := service.ListRecentConversations(ctx, platform, params.Limit)
			if err != nil {
				return errorResult(err.Error()), nil
			}
			return jsonToolResult(items), nil
		case "resolve_recipient":
			item, err := service.ResolveRecipient(ctx, platform, imcore.RecipientLookup{
				Query:          params.Query,
				RecipientID:    params.RecipientID,
				ConversationID: params.ConversationID,
			})
			if err != nil {
				return errorResult(err.Error()), nil
			}
			return jsonToolResult(item), nil
		case "send_message":
			result, err := service.SendMessage(ctx, imcore.SendTarget{
				Platform:       platform,
				RecipientID:    params.RecipientID,
				ConversationID: params.ConversationID,
				Content:        params.Content,
				DryRun:         params.DryRun,
			})
			if err != nil {
				if logger != nil {
					logger.Error("im_api 发送失败", zap.String("platform", params.Platform), zap.Error(err))
				}
				return errorResult(err.Error()), nil
			}
			return jsonToolResult(result), nil
		default:
			return errorResult(fmt.Sprintf("不支持的 im_api action: %s", params.Action)), nil
		}
	})
}
```

## 8. 风险与回滚

### 风险 1: im_api 初期能力不如 feishu_api 完整

缓解：

- 飞书文档/审批/表格不迁移。
- 飞书发送先保留 `feishu_api.send_message` 兼容。

### 风险 2: 平台 hint 误判

缓解：

- 未识别平台时不默认飞书。
- 多平台命中时不授权外发，必须 `question` 澄清或拆成单平台子任务。
- 单平台命中时 `RequiredCapabilitiesForIntent` 使用平台化 capability，错平台工具在 RouteDecision 被拒绝。
- 日志记录 `AllowedDomainsHint`。

### 风险 3: 微信“自动选择联系人”产品语义危险

缓解：

- 只要用户没有明确 recipient/conversation_id，就必须问用户确认。
- 即使只有一个可发送会话也不自动选择。
- 无会话必须提示能力边界。

### 风险 4: 企微/钉钉联系人搜索未接

缓解：

- 明确 unsupported。
- 支持用户提供 conversation_id/chat_id 的发送。
- 后续以 adapter 增量接入通讯录。

### 回滚

- 若 `im_api` 出现问题：
  - 保留 `send_im_message` 和 `feishu_api` 旧入口。
  - 通过 `IM_API_ENABLED=false` 关闭新工具注册。
  - 通过 `IM_API_PREFERRED_OVER_LEGACY=false` 关闭新工具优先召回。
  - 保留平台 hint 和 wrong-platform contract，避免跨平台误判。
- 禁止回滚到“用户说微信但允许 feishu_api 满足”的状态。

## 10. 最终判断

这次不应继续修“哪个词召回哪个工具”。顶层正确抽象是：

```
用户意图
  -> 平台感知 IntentFrame
  -> RouteDecision 按 platform + capability 授权
  -> 模型调用统一 im_api
  -> imcore.Service 选择平台 adapter
  -> adapter 处理平台 ID / owner / tenant / 会话上下文
  -> IntentContract 验证同平台发送结果
```

只有这样，后续新增 Slack、Telegram、邮件、短信、更多企业 IM 时，才是新增 adapter，而不是继续污染主路由、工具召回和契约逻辑。
