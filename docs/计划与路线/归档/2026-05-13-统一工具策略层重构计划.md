**Goal:** 把 builtin、custom、本地 skill、MCP、插件/扩展工具统一到同一套策略层，保证“安全工具默认可用，危险动作才拦截或确认”，不再让 MCP 或任何工具来源拥有各自为战的授权逻辑。

**Architecture:** 所有工具先归一为 `mcphost.ToolDefinition`，再推断成 `router.ToolProfile`，最后由新增的统一策略函数产出 `router.ToolPolicyDecision`。`tool_search`、`RouteDecision`、运行时 admission metadata、ActionGuard 都只消费这个统一决策；MCP annotations、custom tool hints、builtin 表只作为画像输入，不作为单独授权通道。

**Tech Stack:** Go backend (`internal/router`, `internal/master`, `internal/tools`, `internal/mcphost`), table-driven tests, existing `go test ./...`; no frontend change unless final QA reveals displayed copy depends on changed fields.

## 0. 背景与当前问题

当前代码已经有统一承载层：

- `internal/mcphost/host.go` 的 `ToolDefinition` / `ToolExecutor` 承载所有注册工具。
- `internal/router/profile_infer.go` 把 `ToolDefinition` 推断为 `ToolProfile`。
- `internal/router/decision.go` 用 `ToolProfile` 生成 `RouteDecision`。
- `internal/master/action_guard.go` 在执行前做 allow / ask / deny。

但策略层还没有真正统一：

- `TrustedMCPToolProfile`、`trustedMCPArgumentsRequireApproval`、`trusted_mcp_*` reason 让 MCP 成了独立策略分支。
- `tool_search` 自己推 `route_status/callable_now/requires_approval`，没有复用一个统一策略决策。
- `RouteDecision.blockReason` 仍有 trusted MCP 特判。
- ActionGuard 对 shell、external send、structured dangerous、trusted MCP、builtin fallback 分段决策，缺少统一的 profile-first 主路径。
- `ProfileRequiresApproval` 与 `ProfileHasSideEffect` 表达的是粗粒度“可能需要审批”，不是“当前这次调用该 allow/ask/deny”。

用户要求的产品规则：

- 不是 MCP-only，所有工具都要统一。
- 已发现/召回的安全工具应该默认可用。
- 只有危险动作、写入、外发、破坏性、开放世界/runtime exec 才禁止或让用户确认。
- discovery 不是授权，但 discovery 后安全工具进入 RouteDecision 时不能被 blanket `discovery_only/callable_now=false` 卡死。

## 1. 目标行为矩阵

| 工具类型 | 示例 | 默认策略 | 运行时策略 |
|---|---|---|---|
| 只读 builtin | `read_file`, `grep`, `websearch` | 可召回、可路由 | allow |
| 混合 builtin 只读动作 | `memory.search`, `feishu_api.search_docs` | 只读 action 可路由 | allow |
| 混合 builtin 写/外发动作 | `memory.save`, `feishu_api.send_message` | 需要匹配写/外发意图 | ask 或按规则 ask |
| runtime exec | `bash`, `shell`, `run_command` | 默认不进入普通读/写候选；如未来显式 runtime/manage intent 放行，必须先过 capability gate，再由 SafeExecutor 决定 | SafeExecutor allow/ask/deny |
| 破坏性动作 | `delete`, `drop`, `truncate`, `remove_tool` | 默认 blocked 或 ask，按具体风险 | deny 或 ask |
| concurrency-safe custom | `project_status` | 可信本地只读 | allow |
| unknown custom | `opaque_candidate` | fail closed | deny |
| trusted remote/MCP read | `metamcp__query_prometheus` | 可信远端只读 | allow |
| trusted remote/MCP write/send | `metamcp__create_annotation` | 只有副作用意图才路由 | ask |
| trusted remote/MCP destructive/runtime | `metamcp__delete_dashboard`, `metamcp__shell_exec` | blocked dangerous | deny |
| untrusted remote/MCP | `github__create_issue` with no trusted metadata | fail closed | deny |
| skill workflow | `skill-creator`, `mcp-builder` | 只通过 `skill` 入口和 allowed input | route constrained; runtime checks `AllowedToolInputs` |

## 2.1 不在本计划范围

- 不改 MCP transport / handshake / streamable HTTP / SSE 配置层。
- 不改 SafeExecutor 的命令级 allow/ask/deny 规则，只把 shell 工具接入统一 policy 前置判断。
- 不新增 WebUI trusted 开关。
- 不新增新的 tool source 类型，只统一现有 builtin、custom、local skill、MCP/remote、plugin/extension 入口。
- 不改工具召回打分算法，`RecallToolCatalog` 的 score 逻辑保持不变。
- 不做 policy 结果缓存；性能优化作为后续项。
- 不删除 `isExternalSendAction` 的防御性预检查，除非统一 mixed policy 测试证明删除后外发动作仍 ask。

## 3. 设计原则

1. `ToolProfile` 是事实画像，不做最终执行决策。
2. `ToolPolicyDecision` 是策略层唯一输出，所有展示、路由、执行都消费它。
3. source/kind 只能影响 profile inference，不能绕过统一策略。
4. `readOnlyHint=true` 只能降低未知风险，不能覆盖宿主侧危险信号。
5. description/schema/annotations 都是 hint，不是授权。
6. unknown/open-world/destructive 默认 fail closed。
7. side-effect 工具只有在 intent 允许副作用时才可进入 callable path。
8. ActionGuard 的 ask/deny/allow 必须和 RouteDecision 的路由含义一致。
9. Runtime exec 即使 policy 层返回 ask/candidate，也必须继续经过 capability gate 和 SafeExecutor；本计划不把 `bash` 变成普通可召回工具。

```go
package router

import "encoding/json"

type ToolPolicyAction string

const (
	ToolPolicyAllow ToolPolicyAction = "allow"
	ToolPolicyAsk   ToolPolicyAction = "ask"
	ToolPolicyDeny  ToolPolicyAction = "deny"
)

type ToolRouteStatus string

const (
	ToolRouteDiscoveryOnly                ToolRouteStatus = "discovery_only"
	ToolRouteCallableReadOnly             ToolRouteStatus = "callable_read_only"
	ToolRouteCallableWithActionConstraints ToolRouteStatus = "callable_with_action_constraints"
	ToolRouteRequiresSideEffectIntent      ToolRouteStatus = "requires_side_effect_intent"
	ToolRouteRequiresMatchingIntent        ToolRouteStatus = "requires_matching_intent"
	ToolRouteBlockedDangerous              ToolRouteStatus = "blocked_dangerous"
	ToolRouteBlockedUnknown                ToolRouteStatus = "blocked_unknown"
)

type ToolPolicyContext struct {
	Intent    IntentFrame
	Input     json.RawMessage
	ForRoute  bool
	ForAction bool
}

type ToolPolicyDecision struct {
	Action               ToolPolicyAction `json:"action"`
	RouteStatus          ToolRouteStatus   `json:"route_status"`
	CallableNow          bool              `json:"callable_now"`
	RequiresApproval     bool              `json:"requires_approval"`
	RequiresSideEffectIntent bool           `json:"requires_side_effect_intent"`
	Reason               string            `json:"reason,omitempty"`
	Source               string            `json:"source,omitempty"`
	ReadOnly             bool              `json:"read_only,omitempty"`
	SideEffect           bool              `json:"side_effect,omitempty"`
}
```

```go
package router

import (
	"encoding/json"
	"strings"
)

type ToolPolicyAction string

const (
	ToolPolicyAllow ToolPolicyAction = "allow"
	ToolPolicyAsk   ToolPolicyAction = "ask"
	ToolPolicyDeny  ToolPolicyAction = "deny"
)

type ToolRouteStatus string

const (
	ToolRouteDiscoveryOnly                 ToolRouteStatus = "discovery_only"
	ToolRouteCallableReadOnly              ToolRouteStatus = "callable_read_only"
	ToolRouteCallableWithActionConstraints ToolRouteStatus = "callable_with_action_constraints"
	ToolRouteRequiresSideEffectIntent      ToolRouteStatus = "requires_side_effect_intent"
	ToolRouteRequiresMatchingIntent        ToolRouteStatus = "requires_matching_intent"
	ToolRouteBlockedDangerous              ToolRouteStatus = "blocked_dangerous"
	ToolRouteBlockedUnknown                ToolRouteStatus = "blocked_unknown"
)

type ToolPolicyContext struct {
	Intent    IntentFrame
	Input     json.RawMessage
	ForRoute  bool
	ForAction bool
}

type ToolPolicyDecision struct {
	Action                   ToolPolicyAction `json:"action"`
	RouteStatus              ToolRouteStatus   `json:"route_status"`
	CallableNow              bool              `json:"callable_now"`
	RequiresApproval         bool              `json:"requires_approval"`
	RequiresSideEffectIntent bool              `json:"requires_side_effect_intent"`
	Reason                   string            `json:"reason,omitempty"`
	Source                   string            `json:"source,omitempty"`
	ReadOnly                 bool              `json:"read_only,omitempty"`
	SideEffect               bool              `json:"side_effect,omitempty"`
}

func EvaluateToolPolicy(profile ToolProfile, ctx ToolPolicyContext) ToolPolicyDecision {
	// IMPORTANT: mixed 工具的危险动作必须先由 evaluateMixedToolPolicy
	// 转成 ask，而不是被外层 dangerous gate 一刀切 deny。
	// ToolActionProfile 对 StructuredDangerousOperation 返回原 profile，
	// 这个不变量不能随意改；改动前必须先跑 tool_policy_test。
	profile = ToolActionProfile(profile, ctx.Input)
	if isDiscoveryOnlyProfile(profile) {
		return policyDecision(ToolPolicyDeny, ToolRouteDiscoveryOnly, false, false, "discovery_only", profile)
	}
	if profile.Risk == RiskUnknown {
		return policyDecision(ToolPolicyDeny, ToolRouteBlockedUnknown, false, true, "unknown_tool", profile)
	}
	if profile.OpenWorld || profile.Destructive || profile.Risk == RiskDestructive {
		return policyDecision(ToolPolicyDeny, ToolRouteBlockedDangerous, false, true, "dangerous_or_open_world_tool", profile)
	}
	if profile.Risk == RiskRuntimeExec {
		if ctx.Intent.Kind == IntentManageTool && ctx.Intent.AllowsSideEffects {
			return policyDecision(ToolPolicyAsk, ToolRouteRequiresMatchingIntent, true, true, "runtime_exec_under_manage_intent", profile)
		}
		return policyDecision(ToolPolicyDeny, ToolRouteBlockedDangerous, false, true, "runtime_exec_not_allowed", profile)
	}
	if IsMixedReadWriteTool(profile.Name) {
		return evaluateMixedToolPolicy(profile, ctx)
	}
	if profile.Invocation == InvocationSkillTool {
		return policyDecision(ToolPolicyAsk, ToolRouteRequiresMatchingIntent, false, true, "requires_matching_intent", profile)
	}
	if profile.ReadOnly && !ProfileHasSideEffect(profile) && profile.Risk == RiskReadOnly {
		return policyDecision(ToolPolicyAllow, ToolRouteCallableReadOnly, true, false, "read_only", profile)
	}
	if ProfileHasSideEffect(profile) {
		callable := ctx.Intent.AllowsSideEffects && sideEffectIntentAllowsProfile(ctx.Intent, profile)
		decision := policyDecision(ToolPolicyAsk, ToolRouteRequiresSideEffectIntent, callable, true, "side_effect_requires_intent", profile)
		decision.RequiresSideEffectIntent = !callable
		return decision
	}
	return policyDecision(ToolPolicyDeny, ToolRouteBlockedUnknown, false, true, "unclassified_tool_policy", profile)
}

func evaluateMixedToolPolicy(profile ToolProfile, ctx ToolPolicyContext) ToolPolicyDecision {
	if StructuredDangerousOperation(profile.Name, ctx.Input) {
		callable := ctx.Intent.AllowsSideEffects
		decision := policyDecision(ToolPolicyAsk, ToolRouteRequiresSideEffectIntent, callable, true, "structured_dangerous_operation", profile)
		decision.RequiresSideEffectIntent = !callable
		return decision
	}
	if action := structuredAction(ctx.Input); action != "" &&
		containsActionString(ExternalSendActions(profile.Name), action) &&
		!containsActionString(MixedReadOnlyActions(profile.Name), action) {
		callable := ctx.Intent.Kind == IntentExternalWrite && ctx.Intent.AllowsSideEffects
		decision := policyDecision(ToolPolicyAsk, ToolRouteRequiresSideEffectIntent, callable, true, "external_send_action", profile)
		decision.RequiresSideEffectIntent = !callable
		return decision
	}
	if len(MixedAllowedToolInputsForIntent(ctx.Intent, profile.Name)) > 0 {
		return policyDecision(ToolPolicyAllow, ToolRouteCallableWithActionConstraints, true, false, "mixed_action_constraints", profile)
	}
	if len(MixedReadOnlyActions(profile.Name)) > 0 {
		return policyDecision(ToolPolicyAllow, ToolRouteCallableWithActionConstraints, true, false, "mixed_read_actions_available", profile)
	}
	if action := structuredAction(ctx.Input); action != "" {
		return policyDecision(ToolPolicyDeny, ToolRouteBlockedUnknown, false, true, "mixed_action_unknown:"+action, profile)
	}
	return policyDecision(ToolPolicyAsk, ToolRouteRequiresSideEffectIntent, ctx.Intent.AllowsSideEffects, true, "mixed_side_effect_requires_intent", profile)
}

func sideEffectIntentAllowsProfile(intent IntentFrame, profile ToolProfile) bool {
	switch profile.Risk {
	case RiskLocalWrite:
		return intent.Kind == IntentWriteLocal || intent.Kind == IntentExternalWrite
	case RiskExternalWrite:
		return intent.Kind == IntentExternalWrite
	default:
		return intent.AllowsSideEffects
	}
}

		ReadOnly:         profile.ReadOnly,
		SideEffect:       ProfileHasSideEffect(profile),
	}
}
```
