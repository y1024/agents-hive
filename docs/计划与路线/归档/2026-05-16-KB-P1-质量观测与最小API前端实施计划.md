# KB P1 质量观测与最小 API/前端 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

> **状态：已实施并归档。** 2026-05-16 已落地 KB 管理 API、binding/effective bindings、citation 透传、KB quality event、QualityWorkbench `kb_failure_type` 聚合、最小管理页和对话 citation/tool card。最终验证命令见文末“实施验收记录”。

**Goal:** 在 P0 tree-mode 后端闭环基础上，交付 KB 管理 API、binding 管理 API、质量事件、Trace/Replay/QualityWorkbench 聚合，以及对话面 citation 最小可用 UI。

**Architecture:** P1 不扩展 ingest 文件格式，不做客服状态机。后端复用现有 `internal/api` 路由、中间件、auth、admin handler 风格；binding 管理复用 P0 `KBBindingResolver` 和 `kb_bindings`，不把 namespace 选择暴露给普通用户。质量数据复用 `agentquality.Event.Attributes`、`observability` label sanitizer、`qualityworkbench` dashboard/replay 结构。citation 由服务端从 P0 evidence ledger 汇总后写入 assistant message metadata，并透出到 WebSocket 与历史消息 API。前端复用 `NodeClient` / `LocalNodeClient`、zustand store、`ToolAdapter`、`MessageBubble`、i18n 和 admin shell，只加最小 KB 页面、binding 管理、tool card、citation 渲染。

**Tech Stack:** Go、React、TypeScript、Vite、zustand、lucide-react、现有 API middleware、agentquality、observability、qualityworkbench。

## 1. 范围边界

### 做

- Admin API：namespace/document/tree/node/evidence 基础接口。
- Binding API：把 namespace 绑定到 agent/domain/session template/session/tenant/user，并查看 effective bindings。
- Ingest API：只接 markdown 文本或 `.md/.markdown` 上传。
- Quality event：KB retrieval、evidence violation。
- Observability：trace view 展示 KB tool spans，metrics label 低基数。
- QualityWorkbench：聚合 KB 失败类型。
- Chat UI：KB tool 卡片、citation 卡片、无证据警告。
- Admin UI：最小 namespace/doc/tree 只读管理面 + markdown 上传 + 绑定到 Agent/业务域/会话模板。

### 不做

- 不做 PDF/DOCX。
- 不做图片 asset。
- 不接受含本地图片、data URI 图片或需要下载远程图片的 Markdown 静默 active；资产层未启用时必须拒绝或 degraded。
- 不做完整版本 diff。
- 不做客服 webhook/outbox。
- 不做 vector/hybrid。
- 不做移动端管理体验。

## 3. API 设计

P1 只提供最小管理与只读接口：

| 方法 | 路径 | 用途 |
|---|---|---|
| `GET` | `/api/v1/kb/namespaces` | 列出当前 owner/domain 可见 namespace |
| `POST` | `/api/v1/kb/namespaces` | 创建 namespace |
| `GET` | `/api/v1/kb/namespaces/{id}/documents` | 列出文档 |
| `POST` | `/api/v1/kb/namespaces/{id}/documents:ingest-markdown` | 上传/提交 markdown 并 ingest |
| `GET` | `/api/v1/kb/documents/{id}/tree` | 获取剥离 text 的 tree |
| `GET` | `/api/v1/kb/documents/{id}/nodes/{node_id}` | 获取单节点详情，仅 admin 管理面用 |
| `POST` | `/api/v1/kb/documents/{id}:archive` | 归档文档 |
| `GET` | `/api/v1/kb/evidence?session_id=&turn_id=` | 查询本轮 evidence，用于调试和 replay |
| `GET` | `/api/v1/kb/bindings` | 列出 namespace bindings |
| `POST` | `/api/v1/kb/bindings` | 创建 namespace binding |
| `PATCH` | `/api/v1/kb/bindings/{id}` | 启用、禁用或更新 binding |
| `DELETE` | `/api/v1/kb/bindings/{id}` | 删除或禁用 binding，首发建议软删除/disable |
| `GET` | `/api/v1/kb/effective-bindings?agent_id=&domain_id=&session_template_id=&session_id=` | 预览 resolver 结果 |

规则：

- 所有 API 从 auth/session context 推导 owner，不信任 body 里的 owner。
- body 可传 domain/namespace/binding target，但必须通过 domainpolicy、KB store 和 resolver 过滤。
- binding API 不接受 body 里的 owner 字段；`binding_type` 只允许 `agent/domain/session_template/session/tenant/user/system` 中 P1 明确支持的子集。
- 普通用户聊天 API 不暴露 namespace 选择；namespace 只在 admin/debug UI 中可见。
- 未授权文档返回 404，不暴露存在性。
- markdown 上传大小 P1 限制为 2MB，超限 413。
- P1 `documents:ingest-markdown` 只接受纯文本 Markdown；检测到 `![...](./local)`、`![...](data:image/...)` 时返回 422 或 degraded preview，不能生成 active document；远程图片默认保留但标记 degraded，P2 再归档。
- `internal/api.Server` 没有注入 KB service 时，管理 API 返回 503；不要在 handler 内自行 new store/service。
- `GET /api/v1/kb/evidence` 仅面向同 owner/session 的调试/replay，默认不返回原文全文。
- `GET /api/v1/kb/effective-bindings` 只返回 namespace id/name/绑定来源摘要，不返回文档原文；用于管理员预览 Agent 会看到哪些知识。
