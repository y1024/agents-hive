## 背景与目标

### 现状问题

现有系统支持两种个人微信协议：

| 协议 | 实现 | 问题 |
|------|------|------|
| WeChatPadPro | HTTP+WebSocket，连接本地客户端 | 第三方逆向协议，封号风险未知 |
| Wechaty (gRPC) | 连接外部 Wechaty Puppet Service | 同上，且需要额外维护进程 |

两种协议都是**第三方逆向工程**，依赖非官方接口，协议稳定性无法保证。

### wechatbot.dev 是什么

微信官方推出的 **iLink Bot SDK**，支持 Node.js / Python / Go / Rust 四种语言：
- GitHub: `github.com/corespeed-io/wechatbot`
- Go SDK: `go get github.com/corespeed-io/wechatbot/golang`
- 协议：iLink Bot API（长轮询 + QR码登录 + AES-128-ECB CDN加密）
- 特点：会话自动恢复、凭证持久化、官方背书

**Go SDK 能力边界（v2.1.0）：**

| 能力 | 支持 |
|------|:----:|
| 文本/图片/语音/文件/视频消息收发 | ✅ |
| 扫码登录 + 会话持久化 | ✅ |
| 输入状态 (typing indicator) | ✅ |
| CDN 媒体上传/下载 | ✅ |
| 会话自动恢复 (-14 重登录) | ✅ |
| 联系人管理 | ❌ |
| 群管理 | ❌ |
| 朋友圈 (Moments) | ❌ |
| 名片/转发/撤回 | ❌ |

### 重构目标

1. **替换协议**：wechatbot.dev 替代 WeChatPadPro + Wechaty，成为唯一微信协议
2. **多租户**：每个登录用户拥有独立的 WeChat 实例，凭证按用户隔离
3. **消息路由**：用户 A 的微信消息 → 用户 A 的 Agent Session
4. **平滑迁移**：无老用户负担，直接重新配置

## 多租户架构

### 核心设计原则

- **每个用户 = 一个 WeChat 实例**：用户通过扫码登录自己的微信账号
- **凭证隔离**：每个用户的 WeChatBot credentials 存储在独立路径
- **消息路由**：微信发送者 wxid → 映射到系统用户 → 路由到对应 Session
- **权限隔离**：用户只能在自己的微信通道中收发消息

### 消息流详解

#### 接收消息（Inbound）

```
微信用户A ──▶ wechatbot SDK (长轮询) ──▶ Backend.onMessage()
      │
      ▼
IncomingMessage{wxid, content, type}
      │
      ▼
enrichCtx(ctx, wxid, "wechatbot")
      │  查找 external_id=wxid, provider_type=wechatbot 的系统用户
      ▼
ctx with user
      │
      ▼
Router.processMessage()
      │
      ├──▶ sessionID = "im-wechatbot-{wxid}"
      │
      ▼
Master.ProcessMessage(ctx, sessionID, content)
      │
      ▼
回复 content ──▶ Router ──▶ ChannelManager.findBotByWxid(wxid)
      │
      ▼
Bot.SendText(toWxid, content)
```

#### 发送消息（Outbound via 工具）

```
Agent 调用 send_im_message(platform="wechatbot", chat_id=wxid, content="...")
      │
      ▼
IMRouter.SendMessage(ctx, "wechatbot", userID, wxid, content)
      │  ← userID 从 ctx 取得，防止跨用户发送
      ▼
ChannelManager.findBotByWxid(userID, wxid)
      │  ← 校验 bot.OwnerUserID == userID，不匹配则拒绝
      ▼
Bot.SendText(toWxid, content)
```

### 凭证存储结构

```
~/.claw/wechat/
├── {userID-1}/
│   └── credentials.json    # wechatbot SDK 凭证
├── {userID-2}/
│   └── credentials.json
└── ...
```

凭证路径 = `{DataDir}/wechat/{userID}/credentials.json`

### 用户关联机制

已有 `enrichCtx` 机制复用：

- `auth.users` 表有 `external_id` 和 `auth_provider` 字段
- wechatbot 登录后，wxid 作为 external_id，provider_type = `wechatbot`
- 微信消息到达时，`router.processMessage()` 调用 `enrichCtx(ctx, wxid, "wechatbot")` 注入用户上下文
- 会话自动绑定：`sessionID = "im-wechatbot-{wxid}"`

**新问题：如何将 wxid 关联到 system user？**

方案：首次登录时，通过 admin API 或专门的管理端点，将 wxid 绑定到当前登录用户。

### API 设计

### 认证后端

所有 `/api/wechat/*` 接口需要 auth middleware（用户必须先登录系统）。

#### 1. 获取登录状态
```
GET /api/wechat/status
Response:
  logged_out: {
    "status": "logged_out",
    "credentials_exist": true  // 凭证存在但过期，需要重新登录
  }
  logged_in: {
    "status": "logged_in",
    "wxid": "wxid_xxxx",
    "nickname": "张三",
    "avatar_url": "https://..."
  }
  logging_in: {
    "status": "logging_in",
    "qr_pending": true
  }
```

#### 2. WebSocket：核心连接（登录 + 消息）

WS 连接建立后，后端根据当前登录状态推送不同事件：

```
WS /api/wechat/events

Server → Client（未登录）:
  {type: "qr", qr_data: "base64..."}       // 二维码图片 Base64，前端直接渲染 <img>
  {type: "scanned"}                        // 有人扫码了
  {type: "expired"}                         // 二维码过期
  {type: "error", message: "..."}          // 错误

Server → Client（已登录）:
  {type: "login", wxid: "...", nickname: "..."}  // 登录成功
  {type: "logout"}                               // 登出
  {type: "message", wxid: "...", text: "...", msg_id: "...", timestamp: "..."}  // 新消息
  {type: "error", message: "..."}                // 错误

Client → Server:
  (目前无需客户端主动发消息，登录由 SDK 自动管理)
```

#### 3. 获取最近联系人
```
GET /api/wechat/contacts?limit=20
Response: {
  "contacts": [
    {
      "wxid": "wxid_lisi",
      "nickname": "李四",
      "avatar_url": "https://...",
      "last_message": "你好",
      "last_time": "10:30"
    }
  ]
}
```

#### 4. 登出
```
POST /api/wechat/logout
Response: {message: "已退出登录"}
```

#### 5. 强制重新登录（重新扫码）
```
POST /api/wechat/relogin
Response: {message: "正在重新获取二维码"}
→ 触发 WS 推送 {type: "qr", qr_data: "..."}
```

## 已知问题与开放点

### 1. wxid ↔ system user 绑定

**结论：自动绑定。**

设计：用户必须**先登录系统**（auth），再访问 `/wechat` 进行扫码。扫码登录成功后，backend 直接用当前 auth session 的 userID 创建 `ChannelManager` 实例，**无需显式绑定**。

```
用户登录系统 (auth) → 访问 /wechat → 扫码 → SDK 返回 wxid
      │
      ▼
后端: userID = authContext.UserID
      wxid = SDK 返回
      │
      ▼
自动调用 authEngine.BindExternalID(userID, "wechatbot", wxid)
      │
      ▼
写入 auth.users 表:
  external_id = wxid
  auth_provider = "wechatbot"
```

**前端提示**：登录成功后自动跳转到联系人列表，无额外确认步骤。

### 2. 多设备支持

一个微信账号同一时间只能在一个设备登录。wechatbot SDK 内部处理设备冲突（自动踢出旧设备）。但多用户场景下，每个用户用自己的微信账号，不冲突。

### 3. 会话持久化

wechatbot SDK 支持会话恢复（凭证过期自动重登录）。凭证存储在 `{DataDir}/wechat/{userID}/credentials.json`。

### 4. 群聊消息

**结论：Phase 1 不支持群聊，标记为 TODO。**

Phase 1 只支持私聊（1对1）。群聊路由更复杂：
- `room_id` 不对应单一用户
- enrichCtx 无法做用户关联
- 群成员列表需要额外 API（wechatbot SDK 不支持联系人管理）

Phase 2+ 考虑：群聊消息 → 识别群内 @机器人 → 单独路由到 Agent Session。

### 5. 凭证目录权限

`~/.claw/wechat/{userID}/` 需要 `0700` 权限，防止其他用户读取他人凭证。backend 初始化时创建目录并设置权限。

### 6. Session ID 命名空间

微信联系人会话的 Session ID 格式：`im-wechatbot-{wxid}`

每个用户的 wxid 全局唯一，Session ID 不会冲突。

## 错误处理与故障恢复

### SDK 事件 → 系统错误映射

| SDK 事件 | 系统行为 | 用户可见影响 |
|---------|---------|------------|
| `OnError(err)` | log.Error + 记录错误次数 | 静默重试，达到阈值后推送 WS `{type:"error"}` |
| `OnExpired` | 自动请求新二维码，推送 WS `{type:"expired"}` → 立即推送 `{type:"qr"}` | 前端刷新二维码 |
| SDK 长轮询超时 | 内部自动重试，间隔 3s/5s/10s/30s 退避 | 无感知 |
| 网络中断 >5min | 触发凭证重连流程 | WS 推送 `{type:"reconnecting"}` |
| 凭证失效 | 自动登出，推送 WS `{type:"logout", reason:"credential_expired"}` | 页面提示重新扫码 |
| 多设备踢出 | 同上 | 同上 |

### 错误分类

```
CRITICAL（需人工介入）:
  - 凭证文件损坏 → 推送"登录已过期，请重新扫码"
  - SDK 初始化失败（配置错误） → 日志 + admin alert
  - 长期断连（>30min）→ 推送 reconnecting 状态

RETRYABLE（自动恢复）:
  - 长轮询超时 → 内部重试，退避策略
  - 临时网络抖动 → 自动重连
  - 二维码过期 → 自动刷新新二维码

REPORTED（通知用户）:
  - 微信账号被踢出 → 推送 logout 事件
  - 消息发送失败 → 返回错误给 Agent
```

### ChannelManager 并发安全

```
- ChannelManager 使用 sync.RWMutex 保护 map
- 每个 BotInstance 在独立 goroutine 中运行 Run()（长轮询）
- BotInstance.Stop() 通过 context cancel 实现
- 同一用户的并发操作（登录/登出）通过 mutex 串行化
```

### 消息处理重试

```
Inbound 消息处理失败:
  1. 记录错误日志（包含 wxid, content, error）
  2. 不重试（消息已收到，处理失败不应重复投递）
  3. 回复错误给发送者（如果可能）

Outbound 消息发送失败:
  1. sendim 工具层捕获错误
  2. 返回错误信息给 Agent："消息发送失败: {reason}"
  3. 日志记录（包含 from_user, to_wxid, content, error）
```

## 安全威胁模型

### 攻击面分析

| 新攻击面 | 威胁 | 缓解措施 | 严重度 |
|---------|------|---------|--------|
| 凭证文件读取 | 其他用户通过 SSRF/文件读取漏洞获取他人微信凭证 | 凭证目录 0700 权限，Web 服务以非 root 运行 | High |
| WS 连接劫持 | 攻击者伪造 auth token 连接他人 WS | WS 握手时验证 userID 与 auth session 一致 | High |
| 消息注入 | 攻击者伪造消息通过 WS 推送给受害者 | WS 只读（服务端推送），无客户端写入路径 | Low |
| wxid 枚举 | 通过遍历 session ID 访问他人会话 | Session ID 基于 wxid（攻击者不知道），需先知道 wxid | Low |
| 凭证钓鱼 | 攻击者伪造二维码（中间人替换 QR URL） | SDK 返回的 QR URL 直接传给前端，不经过代理 | Medium |
| 媒体文件注入 | 微信收到的媒体文件含恶意 payload | Agent 只处理文本回复，不执行下载的媒体文件 | Low |

### 输入验证

```
wxid 格式: 微信 ID 格式固定，验证 ^[a-zA-Z0-9_]{5,}$ 不匹配 → 拒绝
消息内容: 无限制（AI 需要处理任意输入）
文件上传: wechatbot SDK 内部处理，前端无直接上传路径
```

### 凭证隔离

```
目录结构: ~/.claw/wechat/{userID}/credentials.json
创建时: os.MkdirAll + os.Chmod(0700)
读取时: 验证 file.Uid == process.Uid
Web 服务不应以 root 运行（否则 0700 无法防护）
```

## 附录：wechatbot Go SDK 关键接口

```go
// 安装
go get github.com/corespeed-io/wechatbot/golang

// 核心类型
type IncomingMessage struct {
    UserID        string
    Text          string
    Type          ContentType  // "text", "image", "voice", "file", "video"
    Timestamp     time.Time
    Images        []ImageContent
    Voices        []VoiceContent
    Files         []FileContent
    Videos        []VideoContent
    QuotedMessage *QuotedMessage
}

type Options struct {
    BaseURL  string  // 默认 ilinkai.weixin.qq.com
    CredPath string  // 凭证路径，默认 ~/.wechatbot/credentials.json
    LogLevel string  // debug/info/warn/error/silent
    OnQRURL  func(url string)
    OnScanned func()
    OnExpired func()
    OnError   func(err error)
}

// 核心方法
bot := wechatbot.New(opts)
bot.Login(ctx, false)                    // false = 有凭证则跳过扫码
bot.OnMessage(func(msg *IncomingMessage) {
    bot.SendTyping(ctx, msg.UserID)
    bot.Reply(ctx, msg, "回复内容")
})
bot.Run(ctx)                             // 启动长轮询（阻塞）
bot.Stop()

// 发送（直接）
bot.Send(ctx, userID, "内容")

// 媒体
bot.ReplyContent(ctx, msg, wechatbot.Content{Image: data})
bot.Download(msg)                         // 下载消息中的媒体
bot.Upload(ctx, data, "image/png")        // 上传到 CDN
```
