# 记忆系统优化 & AI 自我进化设计方案

## 背景

当前记忆系统已具备基础架构：FTS5 全文搜索、向量语义搜索、compaction 触发提取、注入 system prompt。
但存在四个明显问题，且完全缺失"自我进化"能力。

---

## 一、现有问题诊断

### 问题 1：提取逻辑脆弱（`internal/memory/extractor.go`）

`parseFacts()` 只解析 `- ` 或 `* ` 开头的 bullet point，用关键词匹配分类。
LLM 摘要格式稍有变化就提取失败。本质上是 grep，不是 AI。

### 问题 2：`feedback` 类型是空壳

`MemoryTypeFeedback` 定义在 `types.go`，但 `Extractor` 没有任何逻辑生成它。
用户纠正 AI、否定工具调用、表达偏好，这些最有价值的信号完全丢失。

### 问题 3：无记忆衰减机制

`MemoryRecord` 有 `AccessCount` 和 `AccessedAt` 字段，但搜索和注入时没有用它们做权重或淘汰。
旧记忆和新记忆权重相同，时间越长噪音越多。

### 问题 4：注入无相关性过滤（`internal/memory/injector.go`）

`SearchOptions.MinScore` 字段定义了但 `InjectContext()` 没有传入。
低质量、低相关性记忆也会被注入，污染 LLM 上下文。

---

## 二、自我进化设计

### 核心思路

自我进化 = AI 从每次对话里学到"下次怎么做得更好"，并真正改变下次行为。

现有系统只记住了"发生了什么"（事实记忆），缺少"哪里做得不好"和"应该怎么改"（行为反思）。

### 三层进化架构

```
第一层：会话级反思（Session Reflection）
    每次会话结束 → 反思 agent 分析对话 → 写入 feedback 记忆

第二层：跨会话模式识别（Pattern Recognition）
    每周批处理 → 聚合同一用户的 feedback → 识别重复模式 → 生成高权重记忆

第三层：行为注入（Behavior Injection）
    每次对话开始 → feedback 记忆优先注入 system prompt 最前面
```

## 三、实现计划

### P0：立即可做（今天）

#### 3.1 修复 MinScore 过滤

**文件：** `internal/memory/injector.go`

`InjectContext()` 搜索时加入 `MinScore` 阈值，过滤低相关性记忆。

```go
result, err = inj.store.Search(ctx, SearchOptions{
    Query:    userMessage,
    Limit:    inj.topK,
    UserID:   userID,
    MinScore: 0.3, // 过滤相关性低于阈值的记忆
})
```

#### 3.2 会话结束触发反思

**触发点：**
- `handleDeleteSession`（`internal/api/session_handlers.go`）：删除前异步触发
- 会话超时检测：`startBackgroundSync` 里检测 30 分钟无活动的会话

**反思 agent prompt：**

```
分析以下对话，提取用户的行为偏好和 AI 的改进点。
输出格式为 JSON 数组，每条记忆包含 type 和 content：
- type: "feedback" 表示 AI 需要改进的行为
- type: "user" 表示用户偏好

重点关注：
1. 用户纠正了什么（"不对"、"应该"、"不要"）
2. 用户明确表达的偏好（语言、风格、详细程度）
3. 哪些工具调用被用户否定或要求重做
4. 本次任务是否完成，卡在哪里
```

**新增文件：** `internal/memory/reflection.go`

```go
type ReflectionAgent struct {
    llmClient *llm.Client
    store     MemoryStore
    logger    *zap.Logger
}

// ReflectOnSession 异步分析会话，提取 feedback 记忆
func (r *ReflectionAgent) ReflectOnSession(ctx context.Context, sessionID, userID string, messages []llm.MessageWithTools) error
```

### P1：本周内

#### 3.3 feedback 记忆优先注入通道

**文件：** `internal/memory/injector.go`

在 `InjectContext()` 里，先单独查询 `MemoryTypeFeedback` 类型的记忆，注入到 system prompt 最前面，再注入普通记忆。

```
## 关于你的工作方式
- 这个用户偏好直接给代码，不要先解释
- 用户使用中文，回复也用中文
- 修改并发相关代码时注意竞态问题（上次 session_loop.go 被指出）

## 相关记忆
- [project] 修复了会话隔离 bug，user_id 为空的会话不再对普通用户可见
- ...
```

#### 3.4 记忆衰减权重

**文件：** `internal/memory/pg_store.go`

搜索时加入时间衰减因子，最近 7 天的记忆权重 ×1.5，超过 90 天的记忆权重 ×0.5。
同时每次记忆被注入时更新 `accessed_at` 和 `access_count`。

---

### P2：下周

#### 3.5 用 LLM 替换 bullet-point 提取

**文件：** `internal/memory/extractor.go`

`ExtractFromSummary()` 改为调用 LLM 提取，而不是 grep bullet points。
降级策略保留：LLM 失败时回退到现有的关键词匹配。

#### 3.6 跨会话模式识别批处理

**新增文件：** `internal/memory/pattern_job.go`

每周跑一次，对同一用户的所有 feedback 记忆做聚合分析：
- 相同主题出现 3 次以上 → 合并为一条高权重记忆
- 过时记忆（90 天未访问）→ 软删除

## 四、数据流全景

```
用户发消息
    │
    ▼
Injector.InjectContext()
    ├── 1. 查 feedback 记忆（优先，注入最前）
    └── 2. 查相关事实记忆（MinScore 过滤后注入）
    │
    ▼
LLM 处理
    │
    ▼
消息积累 → Compaction 触发
    ├── LLMSummaryCompactor（LLM 生成 JSON 摘要）
    └── SessionMemoryCompactor（规则提取会话记忆）
    │
    ▼
Extractor.ExtractFromSummary()（LLM 提取，降级到关键词）
    └── 写入 project/user/reference 记忆
    │
    ▼
会话结束（删除 or 超时）
    │
    ▼
ReflectionAgent.ReflectOnSession()（异步）
    └── 写入 feedback 记忆
    │
    ▼
每周批处理
    └── PatternJob：聚合 feedback，淘汰旧记忆
```

---

## 五、不做什么

- 不做跨用户的记忆共享（隐私边界）
- 不做记忆的手动编辑 UI（先让自动化跑起来再说）
- 不做实时的模式识别（批处理够用，实时成本高）

---

## 六、验收标准

| 功能 | 验收条件 |
|------|---------|
| MinScore 过滤 | 注入的记忆相关性评分均 ≥ 0.3 |
| 会话反思 | 每次会话结束后 DB 里有对应的 feedback 记忆 |
| feedback 优先注入 | system prompt 里 feedback 记忆在普通记忆之前 |
| 记忆衰减 | 90 天未访问的记忆不出现在 top-K 搜索结果里 |
| LLM 提取 | 非 bullet-point 格式的摘要也能提取到记忆 |
